## 실행 순서

1. 완료: Codex가 로컬 테스트를 새 UX 기준으로 통과시켰다.
2. 완료: Codex가 프론트 build 후 7010 로컬 서버를 재시작했다.
3. 다음: TJ님은 이 문서의 `TJ님이 직접 해야 할 일` 1~4번을 먼저 결정한다.
4. Codex는 승인된 상태값과 운영 기준을 admin UI와 문서에 반영한다.
5. 30일 병행 운영을 시작하고, 매주 native lead와 아임웹 fallback 누락률을 확인한다.
6. Go 기준을 만족하면 아임웹 입력폼 축소 또는 redirect 전환을 검토한다.

## 지금 막히는 지점

운영 기준은 아직 0%다. 이유는 코드가 부족해서가 아니라, 운영 secret, 실제 상담원 상태값, 아임웹 export, 첫 랜딩 노출 방식이 아직 확정되지 않았기 때문이다.

Codex와 Claude Code는 로컬 구현과 검증을 계속 진행할 수 있다. 다만 위 1~4번 결정 없이는 운영 반영 판단이 숫자로 닫히지 않는다.
