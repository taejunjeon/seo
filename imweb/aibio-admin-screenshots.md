## 자체 개발에 포함할 기능

| 기능 | 이유 | 우선순위 |
|---|---|---|
| 공개 홈페이지/랜딩 | 현재 유입과 전환의 입구 | 높음 |
| 프로그램/기기/후기/위치 페이지 | 현재 메뉴 IA의 핵심 | 높음 |
| 입력폼/상담예약 | 콘텐츠 반응의 핵심 데이터 | 높음 |
| 카카오/채널톡 CTA | 상담 전환 보조 | 높음 |
| 인기 페이지/유입 도메인 통계 | 아임웹 통계의 대체 최소 범위 | 중간 |
| 첫방문 혜택/오퍼 코드 | 쿠폰 전체보다 운영 목적에 맞음 | 중간 |
| 예약금/체험권 결제 | 결제 자체화 검증 | 중간 |
| 운영자 권한/RBAC | 고객/리드 개인정보 접근 통제 | 중간~높음 |

## 자체 개발에서 제외할 기능

| 기능 | 제외 이유 |
|---|---|
| 아임웹 노코드 디자인 빌더 | 내부 서비스에 불필요하고 개발 범위가 큼 |
| 쇼핑몰 전체 관리자 | AIBIO는 리드/예약 중심 |
| 상품 리뷰/문의 | 현재 화면 기준 사용 흔적이 약함 |
| 최근 본 상품/구매 추천/무료배송 | 물류형 커머스 기능 |
| 정기구독/배송/송장 | AIBIO 센터 홈페이지 MVP와 무관 |
| 전체 쿠폰/포인트 엔진 | 첫방문 혜택만으로 충분 |

## 개발 Phase에 주는 의미

1. 먼저 홈페이지/랜딩과 입력폼을 자체화한다.
2. 그다음 리드 상태, 예약 상태, 방문 상태를 관리자에서 보게 한다.
3. 이후 아임웹 통계 중 필요한 것만 GA4/서버 이벤트로 대체한다.
4. 결제는 예약금/체험권 하나로만 붙인다.
5. 아임웹 디자인 모드와 관리자 전체 복제는 하지 않는다.
