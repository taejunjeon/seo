# 새 광고 계정 / 두 번째 Pixel 판단 업데이트

작성 시각: 2026-05-16 01:51 KST

## 판정

**지금은 새 광고 계정과 두 번째 Pixel을 만들지 않습니다.**

현재 문제는 “Meta가 구매 신호를 전혀 못 받는다”가 아닙니다. 7일 Ads Manager 구매는 184건 있고, Server CAPI도 살아 있습니다.

문제는 2026-05-15 단일일의 Ads purchase 0과 데이터 공유 제한 경고입니다. 이 상태에서 새 계정/새 Pixel을 만들면 원인 분리가 더 어려워질 수 있습니다.

## 새 계정이 지금 위험한 이유

1. 같은 `biocom.kr` 도메인을 쓰면 건강/웰빙 제한이 다시 붙을 가능성이 높습니다.
2. 기존 Pixel 학습 데이터가 쪼개집니다.
3. 광고 계정/Pixel을 바꾸면 “제한 때문인지, 새 계정 학습 부족 때문인지” 구분이 어려워집니다.
4. 단기 ROAS 비교가 깨집니다.
5. 운영 리스크가 큽니다.

## 두 번째 Pixel이 지금 위험한 이유

1. 같은 사이트에 Pixel을 두 개 넣으면 중복 이벤트와 dedup 설계가 복잡해집니다.
2. Purchase 중복 위험이 커집니다.
3. Meta가 같은 도메인/같은 사업 카테고리를 다시 제한할 수 있습니다.
4. 데이터가 분산돼 기존 캠페인 학습이 약해질 수 있습니다.

## 검토 조건

새 광고 계정 또는 두 번째 Pixel은 아래 조건이 모두 쌓일 때만 다시 검토합니다.

1. 2026-05-15 이후 48~72시간이 지나도 Ads Manager purchase가 계속 0이다.
2. Meta UI 상세에서 Purchase 또는 lower-funnel event 제한이 명시된다.
3. CAPI events_received=1은 계속 성공한다.
4. 내부 Meta evidence 결제완료는 계속 발생한다.
5. clean landing canary에서도 Ads attribution gap이 줄지 않는다.
6. Meta review 또는 category correction이 불가하거나 장기 지연된다.

## 지금 우선순위

1. 2026-05-15 Ads purchase 재조회
2. Meta UI 데이터 공유 제한 상세 확인
3. CAPI payload 데이터 최소화 설계
4. clean landing canary
5. 그래도 안 되면 새 계정/두 번째 Pixel 재검토

## 결론

새 계정/새 Pixel은 “복구책”이 아니라 “마지막 분기 실험”입니다. 지금은 기존 Pixel 안에서 clean landing과 데이터 최소화로 먼저 원인을 줄이는 것이 맞습니다.
