## 3. 숫자의 의미

### 3.1 아임웹 API가 반환한 값

```json
{
  "site": "thecleancoffee",
  "synced": 13336,      // 페이지네이션으로 실제 받은 회원 행수
  "totalCount": 13536   // API가 자체 집계한 전체 회원수 ← 홈페이지 "전체 사용자" 값과 일치
}
```

`totalCount`는 아임웹 백엔드가 `meta.total_count` 필드로 알려주는 값. 홈페이지 관리자 UI의 "전체 사용자" 카운터와 **동일한 출처**로 보이며 **정확히 일치**.

### 3.2 로컬 DB 저장 최종치: 13,338명

동기화 API는 "새로 받은 데이터 13,336건"을 upsert하지만, 이전 동기화 시점에 받았던 회원 중 **이번 API 응답에서 빠진 회원(이전 존재, 현재 리스트 비노출)**도 DB에는 남아 있음. 13,336(이번 응답) + 2(이전 잔존) = **13,338**.

### 3.3 `totalCount` 13,536 vs list 응답 13,336: 200명 갭

아임웹 v2 회원 API는 `totalCount`에는 계정이 존재하는 모든 회원을 포함하지만, `list`에는 **특정 상태(탈퇴·휴면·관리자·블록 등)의 회원을 제외**하고 반환하는 것으로 확인됨. 즉 200명 갭 = 비활성/특수상태 회원.

- 사용자가 본 "일반 회원 13,464" = 13,536 - **72** (관리자·운영자 계정 등 내부 제외 인원 추정)
- API list 응답 13,336 = 13,464 - **128** (추가로 API가 리스트에서 빼는 상태 회원)

## 5. 한계점 (정합성 100% 도달 불가 구간)

### 5.1 일반 회원 13,464 ↔ 로컬 DB 13,338 간 126명 갭

**원인 (추정):**
- 아임웹 `/v2/member/members` API가 list에 포함시키지 않는 상태(휴면·탈퇴·블록·운영자) 중 일부는 관리자 UI에서는 "일반 회원"으로 표기되지만 API list에서 빠짐
- 126명은 정확히 어떤 상태인지 API 응답 스펙만으로는 식별 불가

**해소 시도 가능 경로:**
1. 아임웹 v2 API 문서에서 `/v2/member/members`의 쿼리 파라미터로 `status=all` 또는 `member_type=all` 옵션 존재 여부 확인 필요
2. 아임웹 관리자 UI의 "일반 회원 13,464" 지표가 사용하는 필터 조건(상태값) 확인 필요 — 아임웹 고객지원 문의 또는 관리자 화면 내 툴팁 확인
3. 홈페이지에서 본 분류가 `active + dormant(휴면)`까지 합산한 것일 가능성 있음. 이 경우 휴면 전환 회원을 별도 엔드포인트로 조회해 보강 필요

**운영상 영향:**
- 숫자 126명은 전체의 **0.94%**로 CRM·마케팅 액션에는 사실상 영향 없음
- VIP 프로그램 / 알림톡 / 타겟팅은 "SMS 수신 동의한 7,860명" 기준으로 설계되므로 이 갭은 캠페인 대상 선정에 영향 없음

### 5.2 고유 전화번호 중복 576명

- 같은 전화번호로 여러 회원 코드가 매핑된 경우가 576건 존재
- 가능 원인: 회원 탈퇴 후 재가입 / 가족 공유 / 동의 정책 변경으로 재계정 생성
- 정확한 "개인" 기준 집계는 전화번호 기준이 가장 현실적이지만 576건은 수작업 정리 필요

### 5.3 회원 등급 필드(`member_grade`) 미활용

- 전체 13,338명이 `member_grade=""`로 등록되어 있어 아임웹 네이티브 등급 시스템이 사용되지 않음
- 현재 VIP 전략(`coffee/coffeevip.md`)을 실제 시스템에 반영하려면 이 필드를 사용하거나 별도 CRM DB 테이블에 티어 플래그를 둬야 함

### 5.4 `imweb_orders` 로컬 1,937건의 낮은 커버리지

- 아임웹 주문 API로 동기화된 로컬 `imweb_orders.site='thecleancoffee'` = 1,937건
- PlayAuto 기반 7,035명의 주문 데이터보다 훨씬 적음 → 아임웹 주문 API 동기화가 특정 시기·상태 주문만 가져오고 있을 가능성
- 회원수 정합성의 주 범위는 아니지만, "회원 × 구매 이력" 통합 분석 시 별도 조사 필요 과제

## 6. 완료 선언

- **"전체 사용자 13,536" 정합성: ✅ 완료 (API `totalCount` 100% 일치, 동기화 후 확인 완료)**
- **"일반 회원 13,464" 정합성: ⚠️ 로컬 DB 13,338로 0.94% 오차까지 근접 · 126명 갭은 아임웹 API 스펙 한계로 추가 조사 없이는 해소 불가**

즉시 활용 가능한 회원 DB 기준값:
- **전체 회원 총수: 13,536명 (아임웹 API `totalCount`)**
- **운영 가능 회원(전화번호·마케팅 동의 확인 가능): 13,336명 (로컬 DB 저장)**
- **알림톡 발송 가능(SMS 수신 동의): 7,860명**

이 기준으로 coffeevip 전략의 티어별 대상 인원(§3 `coffeevip.md`)을 재검토해도 큰 변동은 없음. 티어 대상 집계는 "2025년 구매 이력 있는 7,035명" 기반이므로 회원 총수 변경과 독립적.
