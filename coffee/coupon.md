# 더클린커피 쿠폰 CRM 연동 조사 보고서

> **기준일**: 2026-04-04
> **대상**: 아임웹 쿠폰 API + 우리 CRM 연동 가능성

---

## 한 줄 결론

**아임웹 API로 쿠폰을 특정 고객에게 발급하고 사용 추적까지 가능하다.** 쿠폰 생성은 아임웹 관리자에서 하고, 발급/추적은 API로 자동화할 수 있다.

---

## 1. 아임웹 쿠폰 API 현황

### 확인된 엔드포인트

| 메서드 | 엔드포인트 | 용도 | 인증 |
|--------|----------|------|------|
| `GET` | `/promotion/shop-coupon` | 쿠폰 목록 조회 | v1 API Key |
| `GET` | `/promotion/shop-coupon/{code}` | 쿠폰 상세 | v1 API Key |
| `GET` | `/promotion/shop-coupon/{code}/coupon-issue` | 발급/사용 현황 | v1 API Key |
| `GET` | `/promotion/shop-coupon/member/{uid}/coupon-issue` | 회원별 쿠폰 현황 | v1 API Key |
| `POST` | `/promotion/shop-coupon/{code}/issue` | **특정 고객에게 쿠폰 발급** | v1 API Key |

### 없는 것

| 기능 | API 지원 | 대안 |
|------|---------|------|
| 쿠폰 **생성** (create) | ❌ 엔드포인트 없음 | 아임웹 관리자에서 수동 생성 |
| 쿠폰 **수정** (update) | ❌ | 관리자에서 수정 |
| 쿠폰 **삭제** (delete) | ❌ | 관리자에서 삭제 |
| 쿠폰 코드 **자동 생성** | ❌ | 관리자에서 코드 생성 |

## 2. 아임웹 쿠폰 기능 분석 (스크린샷 기반)

### 자동발행 쿠폰 설정

스크린샷에서 확인된 아임웹 쿠폰 자동발행 기능:

**기본 설정:**
- 쿠폰명: "첫구매 할인 쿠폰"
- 발행 대상: 첫 회원가입 / **첫 주문 완료** / 쇼핑 등급 변경 / 생일
- 구매 확정: 직접 입력 20일 후 쿠폰 발급

**혜택 설정:**
- 금액 할인 또는 **비율 할인** (20%)
- 최소 주문 금액: 모든 상품 0원
- 최대 할인 금액: 100,000원
- 중복 할인: 단독으로만 사용 가능
- 쿠폰 적용 범위: 모든 상품 / 특정 카테고리 / 특정 상품

**운영 설정:**
- 만료일: 발급일로부터 N일 (예: 2일)
- 사용 횟수: 제한 1회

**알림 설정:**
- 쿠폰 발급 알림 → **카카오 알림톡 자동 발송** (토글 ON/OFF)
- 발송 시기: 즉시 / 점심 시간(12:00~12:30) / 저녁 시간(18:00~18:30)
- 쿠폰 소멸 알림 → 알림톡
- 앱 푸시 (앱 사용 시)
- **알림톡 발송 가능 건수: 38,299.2건**

### 기존 캠페인 (카카오 메시지 자동화)

| 캠페인 | 발송 대상 | 성공 | 상태 |
|--------|---------|------|------|
| 장바구니+1시간(쿠폰x) | 1,473 | 970 | 실행중 |
| 장바구니+1시간(+3천원 할인쿠폰) | 964 | 649 | 일시정지 |

## 3. CRM 연동 시나리오

### 시나리오: 재구매 유도 쿠폰 발송

```
[1단계] 아임웹 관리자에서 쿠폰 생성
  - 쿠폰명: "30일 미구매 고객 특별 할인"
  - 할인: 15% (최대 ₩10,000)
  - 유효기간: 발급 후 48시간
  - 대상: 지정발행 (API로 발급)
      ↓
[2단계] 우리 CRM에서 재구매 후보 추출
  - GET /api/crm-local/repurchase-candidates?site=thecleancoffee
  - 현재 100명 후보, 22명 SMS 동의
      ↓
[3단계] 아임웹 API로 쿠폰 발급
  - POST /promotion/shop-coupon/{couponCode}/issue
  - 각 고객의 memberUid로 개별 발급
  - 발급 결과를 로컬 DB에 기록
      ↓
[4단계] 알리고로 메시지 발송
  - 알림톡: "쿠폰이 발급되었습니다! 48시간 내 사용 가능"
  - SMS fallback: 알림톡 실패 시 자동 SMS 발송
      ↓
[5단계] 사용 추적
  - GET /promotion/shop-coupon/{code}/coupon-issue → 발급/사용 현황
  - 주문 데이터 imweb_orders.use_issue_coupon_codes 필드로 매칭
  - 쿠폰 사용 매출 = 해당 주문의 payment_amount
```

### 기술적 실현 가능 여부

| 단계 | 가능 여부 | 근거 |
|------|---------|------|
| 쿠폰 생성 | ⚠️ 관리자 수동 | API에 create 엔드포인트 없음 |
| 후보 추출 | ✅ 가동 중 | `/api/crm-local/repurchase-candidates` |
| 쿠폰 발급 | ✅ API 지원 | `POST /promotion/shop-coupon/{code}/issue` |
| 메시지 발송 | ✅ 가동 중 | 알리고 알림톡/SMS + fallback |
| 사용 추적 | ✅ API 지원 | coupon-issue 조회 + 주문 내 쿠폰 코드 |
| ROI 계산 | ✅ 가능 | 쿠폰 할인액 vs 매출 증분 비교 |

## 4. 주의사항

### 아임웹 알림톡 자동 발송과의 중복

아임웹 쿠폰 설정에 "쿠폰 발급 알림 · 알림톡" 토글이 있음.

| 상황 | 아임웹 알림톡 | 우리 알리고 | 고객 경험 |
|------|-----------|-----------|---------|
| 둘 다 ON | 아임웹이 발급 알림 전송 | 우리도 발송 | **중복 수신** |
| 아임웹 OFF + 우리 ON | - | 우리가 발송 | 정상 |
| 아임웹 ON + 우리 OFF | 아임웹이 발송 | - | 정상이지만 우리 추적 불가 |

**권장**: 우리 CRM으로 발송할 때는 아임웹 쿠폰의 "알림톡 자동 발송" 토글을 **OFF**로 설정. 그래야 발송 타이밍/내용/추적을 우리가 제어할 수 있음.

### 자동발행 쿠폰 vs API 발급

- **자동발행**: 아임웹이 조건(첫 주문, 생일 등) 충족 시 자동 발급 + 알림
- **API 발급**: 우리 CRM이 원하는 시점에 원하는 고객에게 발급
- 자동발행 쿠폰도 `GET /promotion/shop-coupon`으로 조회 가능 (coupon code로 식별)
- 하지만 자동발행 조건은 API로 변경 불가 (관리자 전용)

### 인증 방식

- 현재 `IMWEB_API_KEY` + `IMWEB_SECRET_KEY`로 v1 인증 사용 중
- 쿠폰 API가 v1에서 동작하는지 **실제 호출로 확인 필요**
- v2(OAuth) 전용이면 `IMWEB_CLIENT_ID` + `IMWEB_CLIENT_SECRET` 추가 필요

## 6. 쿠폰 ROI 예상

아임웹 기존 캠페인 데이터 기준:

| 지표 | 장바구니 리마인드 (쿠폰x) | 장바구니 리마인드 (+3천원) |
|------|---------------------|---------------------|
| 발송 대상 | 1,473 | 964 |
| 발송 성공 | 970 (66%) | 649 (67%) |
| 구매 전환 (추정) | 25% | 추정 불가 (일시정지) |

**재구매 쿠폰 시나리오 예상:**
- 대상: 30~90일 미구매 고객 중 SMS 동의자 22명
- 쿠폰: 15% 할인 (최대 ₩10,000)
- 전환율 예상: 10~15% (장바구니 리마인드보다 낮지만, 쿠폰 효과로 상승)
- 예상 전환: 2~3명
- 평균 주문액: ₩46,880
- 쿠폰 할인: ₩7,032 (15%)
- 예상 매출: ₩94,000 ~ ₩141,000
- 쿠폰 비용: ₩14,064 ~ ₩21,096
- **순 이익: ₩80,000 ~ ₩120,000**
