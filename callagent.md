# Biocom Growth AI Agent 구상 0329

기준일: 2026-03-29

## 1. 왜 지금 `AI Agent`로 승격해야 하나

지금까지 만든 `callprice` 인사이트는 이미 의사결정 가치가 있다.
문제는 이 구조가 아직 `질문 -> 분석 -> 문서 반영` 중심이라는 점이다.

즉 현재 상태는:

- 사람이 궁금한 질문을 던진다
- 분석가가 데이터를 읽는다
- 해석을 문서나 대시보드에 적는다
- 다음 질문이 나오면 다시 같은 과정을 반복한다

이 방식은 좋은 시작이지만, 아래 한계가 있다.

- 중요한 이상징후를 사람이 질문하기 전까지 놓칠 수 있다
- 매번 같은 join, 같은 해석, 같은 기준선 확인을 반복한다
- CRM, 쿠폰, 상담사 운영, 리드 대기 같은 액션이 분석과 분리돼 있다
- `무엇을 먼저 해야 하는가`가 사람의 감에 많이 의존한다

그래서 다음 단계는 `단발성 분석 대시보드`가 아니라,
**통합적이고 지속적으로 기회를 찾고 액션을 제안하는 그로스 AI Agent**다.

## 2. 대시보드와 AI Agent의 차이

대시보드는 `현재 상태를 보여주는 도구`다.
AI Agent는 `현재 상태를 읽고, 기회를 우선순위화하고, 액션까지 연결하는 운영 레이어`다.

### 대시보드

- 수치를 보여준다
- 사람이 해석한다
- 사람이 우선순위를 정한다
- 사람이 액션을 설계한다

### 그로스 AI Agent

- 매일 데이터를 자동으로 읽는다
- 이상징후와 성장 기회를 먼저 찾는다
- 왜 그런지 원인 후보를 분해한다
- 어떤 액션이 가장 파급력이 큰지 제안한다
- 실행 결과를 다시 학습해 다음 제안을 고도화한다

한 줄로 말하면:

> 대시보드는 `눈`, AI Agent는 `눈 + 두뇌 + 운영 제안 엔진`이다.

## 3. 우리가 만들고 싶은 최종 형태

최종 형태는 아래 5단계가 자동으로 이어지는 구조다.

1. **감지**
   - 전환율 급락
   - 상담 대기 증가
   - 특정 상담사 랜딩 지연
   - 특정 검사 유형 구매 지연
   - 쿠폰 사용률 급등/급락

2. **분해**
   - 리드 질 문제인지
   - 상담사 운영 문제인지
   - 상담 후 CRM 문제인지
   - 쿠폰 설계 문제인지
   - 주말/평일 믹스 문제인지

3. **우선순위화**
   - 영향도
   - 신뢰도
   - 실행 난이도
   - 예상 회수 기간

4. **액션 제안**
   - D+0 / D+3 CRM 변경
   - 상담 후 영양제 쿠폰 실험
   - 정기구독 전환 쿠폰 실험
   - 주말 후속 메시지 실험
   - 수습 상담사 코칭 포인트 제안

5. **학습**
   - 무엇이 실제로 먹혔는지
   - 누구에게 먹혔는지
   - 할인비용 대비 남는 매출이 있었는지
   - 장기 LTR까지 이어졌는지

## 4. 지금 있는 기반

이미 기반은 꽤 있다.

- 상담 원장
- 주문 원장
- LTR 코호트
- 상담사별 성과 비교
- 상담 후 영양제 구매 시점
- 정기구독 전환 흐름
- 상담군 vs 미상담군 차이
- 평일 vs 주말 차이
- 신규 상담사 랜딩 속도

즉 `0에서 시작`이 아니다.
이미 `Agent의 두뇌에 넣을 학습 재료`는 충분히 쌓여 있다.

## 5. 지금 빠져 있는 것

AI Agent로 승격하려면 아래 6개가 더 필요하다.

### 5-1. 고객 타임라인

지금도 전화번호 중심으로 많이 묶을 수 있지만,
장기적으로는 `customer_key` 기준 단일 타임라인이 필요하다.

한 고객에 대해 아래가 한 줄로 이어져야 한다.

- 리드 생성
- 상담 예약
- 슬롯 배정
- 실제 연결
- 상담사
- 검사 유형
- 영양제 첫 구매
- 정기구독 전환
- 쿠폰 발급
- 쿠폰 사용
- 재구매

### 5-2. 상담 대기 로그

지금 가장 큰 블라인드 스팟이다.

필수 필드:

- `lead_created_at`
- `slot_assigned_at`
- `connected_at`
- `lost_reason`
- `reschedule_count`

이게 있어야 `대기 때문에 월 몇 건을 잃는가`를 말할 수 있다.

### 5-3. 쿠폰 정체성

지금은 할인 금액은 일부 보이지만, 쿠폰 목적이 안 보인다.

필수 추적:

- `정기구독 쿠폰`
- `상담 후 영양제 구매 쿠폰`
- 쿠폰 코드
- 쿠폰 이름
- 발급 시각
- 사용 시각
- 상담 후 경과일

### 5-4. 활성 정기구독 상태

지금은 `정기 주문 이력`은 있지만,
`오늘 현재 살아 있는 활성 구독자`를 바로 세기 어렵다.

필요 값:

- 구독 시작일
- 해지일
- 다음 결제 예정일
- 활성 여부

### 5-5. 실험 로그

AI Agent는 추천만 하는 것이 아니라,
`이 액션을 했더니 실제로 어떻게 변했는가`를 기억해야 한다.

필수 값:

- 실험명
- 실험 시작일/종료일
- 대상 조건
- 메시지/쿠폰/오퍼 내용
- 대조군
- 결과 지표

### 5-6. 마케팅 비용 연결

리드 수와 질이 어디서 오는지까지 보려면
메타 광고/랜딩/리드 원천을 붙여야 한다.

그래야 아래가 가능하다.

- 채널별 CAC
- 채널별 상담 전환율
- 채널별 정기구독 전환율
- 채널별 장기 LTR

## 6. 추천 구조: `하나의 이름 + 여러 전문 에이전트`

단일 에이전트보다, `오케스트레이터 1개 + 전문 에이전트 4개`가 운영상 더 좋다.

### 추천 메인 이름

**프로메테우스 (`PROMETHEUS`)**

이유:

- `Forethought`, 즉 미리 내다보는 자라는 뜻이 있다
- 불을 가져온 존재라서 `인사이트를 실제 성장 동력으로 바꾼다`는 상징이 강하다
- `쌉천재 + 미래를 보는 눈 + 실행력` 느낌이 같이 난다

### 추천 서브 에이전트 이름

1. **아테나 (`ATHENA`)**
   - 전략, 분해, 우선순위화 담당
   - “무엇이 가장 큰 문제인가”를 판단

2. **아폴론 (`APOLLO`)**
   - 예측, 추세, 시그널 담당
   - “앞으로 무엇이 일어날 가능성이 큰가”를 본다

3. **플루토스 (`PLUTUS`)**
   - 매출, 풍요, LTR, 구독 담당
   - “무엇이 돈을 남기는가”를 본다

4. **헤르메스 (`HERMES`)**
   - CRM, 메시지, 쿠폰, 실행 전달 담당
   - “누구에게 어떤 액션을 언제 보낼 것인가”를 담당

### 이름 후보를 더 짧게 쓰고 싶다면

- 메인: `프로메테우스`
- 전략 뇌: `아테나`
- 예측 엔진: `아폴론`
- 매출 엔진: `플루토스`

이 4개 조합이 가장 좋다.

## 7. 추천 운영 구조

### Layer 1. Data

원천 데이터:

- 상담 원장
- 주문 원장
- LTR 코호트
- 정기구독 주문
- 쿠폰 마스터 / 쿠폰 사용 로그
- ChannelTalk CRM 이벤트
- Meta 광고 리드/비용
- 리드 대기 이벤트 로그

### Layer 2. Mart

운영용 집계 테이블:

- `consultation_daily_funnel`
- `supplement_purchase_timing`
- `subscription_transition`
- `manager_rampup`
- `coupon_effectiveness`
- `wait_loss_funnel`
- `lead_source_quality`

### Layer 3. Agent Brain

매일 또는 매주 자동으로 수행:

- 이상 탐지
- 기회 탐지
- 원인 후보 분해
- 실험 우선순위 산정
- 예상 uplift 계산

### Layer 4. Action

실제 실행:

- 상담 후 D+0 메시지
- D+1~D+3 리마인드
- 정기구독 쿠폰 발급
- 주말 상담 후 월요일 리마인드
- 장기 미구매 윈백

### Layer 5. Review

주간/월간 리포트:

- 이번 주 가장 큰 기회 3개
- 이번 주 가장 큰 리스크 3개
- 새로 확인된 인사이트
- 실험 결과와 다음 액션

## 9. 지금 당장 가장 파급력 큰 5가지

3. `상담 후 0~3일 미구매 이탈군` 자동 세그먼트화

5. `채널별 리드 질`과 `상담사별 슬롯 생산성` 분리

## 10. TJ님 관점 최종 판단

지금 TJ님이 좋아한 것은 단순히 숫자 몇 개가 아니다.
핵심은 아래다.

- 질문을 던지면 답이 나온다
- 답이 숫자 하나가 아니라 해석까지 붙는다
- 해석이 다음 액션으로 이어진다

이걸 지속 가능하게 만들려면,
`대시보드`를 더 예쁘게 만드는 것보다
**지표 -> 원인 -> 액션 -> 학습**이 계속 이어지는 운영체계를 먼저 만들어야 한다.

그래서 추천 결론은 아래다.

> `callprice`는 하나의 리포트로 끝내지 말고,
> `PROMETHEUS`라는 지속형 그로스 AI Agent의 첫 번째 두뇌 조각으로 승격시키는 것이 맞다.
