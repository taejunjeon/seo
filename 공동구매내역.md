기준 사이트: biocom
문서 성격: 바이오컴 2026년 1~3월 공동구매/파트너 판매 내역 정리

## 10초 요약

2026년 1~3월 바이오컴 공동구매성 주문은 금액 확인 가능한 로컬 정본 기준 1,668건, 512,906,191원이다. 같은 기간 바이오컴 전체 주문 7,358건, 2,088,839,467원 중 주문수 22.7%, 매출 24.6%를 차지했다.

1월은 연유샘, 캥맨, 빵냥 1차가 커서 공동구매 매출 비중이 38.9%까지 올라갔다. 2월은 연뜰살뜰 6차와 송율 7차, 3월은 빵냥 2차와 꾸노핑, 나나베베 3차가 핵심이었다.

주의할 점은 2026-01-01 ~ 2026-01-07 일부 라인아이템 176건은 상품명만 있고 주문 헤더 금액이 없어 매출 합계에서 제외했다. 그래서 이 문서의 금액 합계는 보수적으로 봐야 한다.

## 문서 목적

이 문서는 2026년 1~3월 바이오컴 공동구매가 실제 주문과 매출에서 어느 정도 비중이었는지 한 장으로 확인하기 위한 기준 문서다.

## 이 작업이 하는 일

아임웹 주문 헤더와 PlayAuto 상품 라인을 연결해서, 공동구매성 상품명을 가진 주문을 월별, 파트너별, 차수별로 정리한다.

이 문서는 고객명, 전화번호, 이메일 같은 개인정보를 다루지 않는다. 주문수, 수량, 주문 매출만 집계한다.

## 집계 기준

원천은 로컬 SQLite 파일 `backend/data/crm.sqlite3`이다.

사용 테이블은 두 개다.

| 테이블 | 사용 이유 |
|---|---|
| `imweb_orders` | 주문 헤더, 주문일, 주문 결제금액 확인 |
| `imweb_order_items` | 상품명, 수량, 공동구매 상품명 확인 |

기준은 다음과 같다.

| 항목 | 기준 |
|---|---|
| 사이트 | `site='biocom'` |
| 기간 | `2026-01-01 <= 주문일 < 2026-04-01` |
| 금액 | `imweb_orders.payment_amount` |
| 상품명 | `imweb_order_items.item_name` |
| 넓은 공동구매 기준 | 상품명이 `[`로 시작하고, `정기구독`, `비밀링크`, `KOLAS 인정 획득 기념`을 제외한 파트너/공구성 상품 |
| 좁은 공동구매 기준 | 상품명에 `공구` 또는 `공동구매`가 직접 들어간 상품 |

금액은 라인아이템 금액이 아니라 주문 헤더 금액이다. 현재 `imweb_order_items.pay_amt`는 모두 0원으로 들어와 있어서 상품 라인별 순매출은 바로 계산할 수 없다.

그래서 파트너/차수 단위 집계에서는 같은 주문에 여러 공동구매 상품이 들어 있어도 주문을 한 번만 센다. 예를 들어 김팀장 1차에서 한 주문에 바이오밸런스와 뉴로마스터가 같이 있으면 김팀장 1차 주문 1건으로 본다.

## 현재 상태

확인된 것은 다음과 같다.

- 2026-01-07 이후 주문 헤더와 상품 라인은 대부분 조인된다.
- 2026년 1~3월 공동구매성 주문은 최소 1,668건이다.
- 공동구매성 주문 매출은 최소 512,906,191원이다.
- 전체 주문 대비 공동구매성 주문 매출 비중은 24.6%다.

아직 안 된 것은 다음과 같다.

- 2026-01-01 ~ 2026-01-07 일부 상품 라인은 주문 헤더가 없어 금액을 붙이지 못했다.
- 라인아이템별 실제 결제금액은 없다. 현재는 주문 전체 결제금액을 붙인 값이다.
- 파트너명만 있고 `공구`라는 단어가 없는 항목은 운영 정의 확정이 필요하다.

지금 막힌 이유는 단순하다.

로컬 `imweb_order_items`에는 2026-01-01 ~ 2026-01-07 상품 라인이 일부 존재하지만, 해당 주문번호가 `imweb_orders`의 `biocom` 주문 헤더와 조인되지 않는다. 이 구간은 운영 DB 또는 PlayAuto 원본에서 주문 헤더를 다시 확인해야 금액까지 닫힌다.

- 3월은 전체 매출이 가장 크지만 공동구매 비중은 16.1%로 내려갔다. 빵냥 2차는 컸지만 전체 매출 모수가 더 커졌다.

### 월별 전체 대비 공동구매성 비중 쿼리

```sql
WITH all_orders AS (
  SELECT
    substr(order_time,1,7) AS month,
    COUNT(DISTINCT order_no) AS all_orders,
    SUM(payment_amount) AS all_payment
  FROM imweb_orders
  WHERE site='biocom'
    AND order_time >= '2026-01-01'
    AND order_time < '2026-04-01'
  GROUP BY month
),
offer_items AS (
  SELECT DISTINCT order_no
  FROM imweb_order_items
  WHERE site='biocom'
    AND ord_time >= '2026-01-01'
    AND ord_time < '2026-04-01'
    AND item_name LIKE '[%'
    AND item_name NOT LIKE '[정기구독]%'
    AND item_name NOT LIKE '[비밀링크]%'
    AND item_name NOT LIKE '[KOLAS%'
),
offer_orders AS (
  SELECT
    substr(o.order_time,1,7) AS month,
    COUNT(DISTINCT o.order_no) AS offer_orders,
    SUM(o.payment_amount) AS offer_payment
  FROM offer_items oi
  JOIN imweb_orders o
    ON o.site='biocom'
   AND o.order_no=oi.order_no
  WHERE o.order_time >= '2026-01-01'
    AND o.order_time < '2026-04-01'
  GROUP BY month
)
SELECT
  a.month,
  a.all_orders,
  a.all_payment,
  COALESCE(g.offer_orders,0) AS groupbuy_orders,
  COALESCE(g.offer_payment,0) AS groupbuy_payment,
  ROUND(100.0*COALESCE(g.offer_orders,0)/a.all_orders,1) AS order_share_pct,
  ROUND(100.0*COALESCE(g.offer_payment,0)/a.all_payment,1) AS revenue_share_pct
FROM all_orders a
LEFT JOIN offer_orders g
  ON g.month=a.month
ORDER BY a.month;
```

### 문서 작성 판단

이 문서는 `공동구매`를 회계 계정으로 확정하는 문서가 아니다. 현재 주문 원장에서 관찰되는 공동구매성 판매 흐름을 정리한 운영 기준 문서다.

회계 정산용으로 쓰려면 2026-01-01 ~ 2026-01-07 주문 헤더 복구와 상품별 실결제금액 확보가 먼저 필요하다.
