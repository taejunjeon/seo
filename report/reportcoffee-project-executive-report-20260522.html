O3. Slack 정기 보고 매주·매월 매출, 광고비, 광고비 비중을 Slack으로 받는다. 매번 DB/API를 열지 않고도 운영 판단을 반복해야 한다. no-send JSON -> 사람이 읽는 preview -> 예약 발송 -> 예약 ID 확인 순서로 간다. 100% 정상/경고 메시지가 지정 채널에 정해진 시간에 들어오고, 예약 ID가 기록된다.
O4. 바이오컴 확장 더클린커피에서 검증한 보고 구조를 바이오컴에도 적용한다. 두 브랜드를 같은 숫자 언어로 봐야 예산 배분을 할 수 있다. reportbiocom에 자사몰, NPay, Google, Meta, Naver 데이터 출처와 freshness를 붙인다. 74% 바이오컴도 주간·월간 매출, 광고비, 광고비 비중이 같은 포맷으로 나온다.

단계별 추진계획

  1. 자사몰 기준을 Imweb complete_time 존재 주문으로 전환했다. 무엇을 했는가: Slack no-send JSON의 자사몰 매출 금액을 아임웹 결제/주문 기간 안의 complete_time 존재 주문 기준으로 계산합니다. 왜 하는가: 이 기준이 기존 Excel 금액 5,334,362원과 정확히 맞았기 때문입니다. 어떻게 하는가: Toss+NPay 조합은 결제수단 breakdown 참고값으로 남기고, 자사몰 총액은 Imweb aggregate를 primary로 둡니다.
  2. 스마트스토어는 PlayAuto 경고 포함으로 운영한다. 무엇을 하는가: PlayAuto 1,839,340원 / 53 rows와 Excel 1,905,140원 / 55건의 차이를 Slack no-send에 경고로 표시합니다. 왜 하는가: 운영은 시작해야 하지만, 65,800원 / 2건 차이가 닫히지 않은 숫자를 완전 확정값처럼 보내면 안 되기 때문입니다. 어떻게 하는가: 네이버 직접 API 원장 확보 전까지 PlayAuto를 best available source로 두고, 커머스API 키·IP·스토어 권한이 열리면 no-raw 집계기로 대조합니다.
  3. 네이버 광고비 기간을 매출 기간과 맞춘다. 무엇을 하는가: 2026-05-18 - 2026-05-24 기준 no-send를 새로 만들거나, Hermes에 2026-05-17 - 2026-05-23 XLSX를 다시 요청합니다. 왜 하는가: 광고비와 매출 기간이 하루라도 어긋나면 매출 대비 광고비 비중이 틀어지기 때문입니다. 어떻게 하는가: 검색광고 API 440원과 ADVoost XLSX 350,098원을 같은 기간에서 합칩니다.
  4. 쿠팡 판매 분석 원본을 Hermes로 고정한다. 무엇을 하는가: 쿠팡 판매 분석 화면에서 2026-04-25 - 2026-05-01, 판매자배송+로켓그로스, 커피/차 필터의 기간별·상품별 엑셀을 다시 받습니다. 왜 하는가: 첨부 엑셀은 2,453,600원으로 기존 기준 2,100,400원과 다르고, 상품별 파일이 있어야 60,700원 / 2건 차이를 좁힐 수 있습니다. 어떻게 하는가: Hermes가 브라우저에서 엑셀을 저장하고, Codex가 TeamKeto 상품 353,200원과 남은 60,700원을 분리합니다.
  5. 제품별 TOP 매출을 보고서에 고정한다. 무엇을 하는가: 채널별 총매출 옆에 잘 팔린 상품 TOP3를 붙입니다. 왜 하는가: 매출이 늘거나 줄었을 때 어느 상품 때문인지 바로 봐야 재고와 광고비 판단이 됩니다. 어떻게 하는가: 스마트스토어는 PlayAuto, 쿠팡은 TeamKeto ordersheets, 자사몰은 Imweb order items를 보조 source로 사용합니다.
  6. 바이오컴을 같은 포맷으로 확장한다. 무엇을 하는가: 더클린커피에서 검증한 보고 구조를 바이오컴에 복사합니다. 왜 하는가: 브랜드별 매출과 광고비를 같은 언어로 봐야 예산 배분이 됩니다. 어떻게 하는가: reportbiocom에 source map, no-send JSON, 프론트엔드 보고서를 순서대로 붙입니다.
