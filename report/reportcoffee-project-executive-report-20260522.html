프로젝트를 왜 하는가

지금 필요한 것은 “광고 플랫폼별로 좋아 보이는 숫자”가 아니라, 실제 결제완료 매출을 기준으로 광고비 비중을 보는 운영 보고입니다. NPay 클릭, GA4 purchase, 플랫폼 주장 매출은 참고값일 수 있지만 실제 매출 합계로 바로 쓰면 안 됩니다.

1. 매출 원장 분리 자사몰, 스마트스토어, 쿠팡을 한 줄로 합치지 않고 데이터 출처(source: 숫자가 나온 원천)별로 확인합니다.
2. 광고비 출처 확인 Meta, Naver, Google, TikTok spend를 사이트 기준으로 분리합니다.
3. 발송 전 미리보기 Slack으로 보내기 전에 발송 전 미리보기(no-send preview: 실제 발송 없이 문구와 숫자만 확인)를 먼저 검증합니다.
4. 정기 보고 주간·월간 매출, 광고비, 광고비 비중을 자동 보고합니다.

가장 낮은 Track은 D와 C입니다. D는 바이오컴도 같은 보고 구조로 확장해야 올라가고, C는 더클린커피 Google/TikTok 광고비 매핑과 Naver DB 캐시 저장 안전장치가 남아 있습니다.

  • C를 올리는 일: 네이버 광고비 수집 스크립트에 “더클린커피 캠페인 6개만 읽는 조건”을 넣고, Google/TikTok 캠페인도 브랜드별로 분리한다.
  • D를 올리는 일: 바이오컴도 같은 주간/월간 매출·광고비 양식으로 데이터 출처 지도를 만든다.
  • F를 올리는 일: Slack 발송 전 미리보기를 정기 실행 가능한 스크립트로 묶고, 발송 전 숫자 검증을 자동화한다.

OKR(목표와 핵심 결과)과 액션플랜

OKR 무엇을 달성할 것인가 왜 필요한가 어떻게 진행할 것인가 현재 상태 100% 조건
O1. 실제 매출 기준 고정 채널별 매출을 실제 결제완료 기준으로 분리한다. NPay 클릭이나 플랫폼 주장값이 매출로 섞이면 광고비 비중이 왜곡된다. 자사몰, 스마트스토어, 쿠팡을 데이터 출처별로 나누고 중복 제거 규칙을 문서화한다. 더클린커피 89% 주간·월간 확정 기준 매출이 출처 경고와 함께 자동 산출된다.
O2. 광고비 출처 분리 Meta/Naver/Google/TikTok 광고비를 사이트 기준으로 나눈다. 바이오컴 광고비가 더클린커피에 섞이면 ROAS와 광고비 비중이 무너진다. Meta는 포함하고, Naver는 더클린커피 캠페인 허용 목록을 적용한다. Google/TikTok은 브랜드별 캠페인 매핑 후 포함한다. 더클린커피 62% 각 플랫폼 광고비가 포함, 보류, 제외로 자동 분류된다.
O3. Slack 정기 보고 매주·매월 매출, 광고비, 광고비 비중을 Slack으로 받는다. 사람이 매번 DB/API를 보지 않아도 사업 판단을 반복할 수 있어야 한다. 발송 전 미리보기 -> 승인 -> 실제 채널 발송 -> 로그 확인 순서로 간다. 89% 정상/경고 메시지가 지정 채널에 정해진 시간에 들어온다.
O4. 바이오컴으로 확장 더클린커피에서 검증한 보고 구조를 바이오컴에도 적용한다. 두 브랜드를 같은 숫자 언어로 봐야 예산 배분 판단이 가능하다. reportbiocom에 같은 채널, 데이터 출처, 최신성 구조를 붙인다. 20% 바이오컴도 주간·월간 매출, 광고비, 광고비 비중이 같은 포맷으로 나온다.

보고에는 0원 후보, 캐시 저장은 보류

Slack 보고에서는 Naver를 0원 후보로 표시할 수 있습니다. 하지만 DB 캐시 저장(cache write: 화면이 빠르게 읽도록 내부 SQLite 테이블에 저장)은 “더클린커피 캠페인 6개만 저장하는 안전장치”를 광고비 수집 스크립트에 넣은 뒤 별도 승인으로 진행해야 합니다.

정확히 어디를 고치는가

다음 작업은 네이버 광고 계정 자체를 바꾸는 일이 아닙니다. 광고를 켜거나 끄지 않고, 광고비를 읽는 내부 수집 경로에 안전장치를 붙이는 일입니다.

1. Naver Ads API 네이버 광고 계정에서 캠페인 이름, 상태, 광고비를 읽는 공식 통로입니다. 여기에는 쓰기나 상태 변경을 하지 않습니다.
2. 광고비 수집 스크립트 `backend/scripts/naver-ads-collect-7d-20260513.ts`가 네이버 광고비를 읽어 내부 DB에 저장할 수 있는 프로그램입니다. 여기에 “더클린커피 후보 6개만 통과” 조건을 넣어야 합니다.
3. DB 캐시 테이블 `naver_ads_daily`는 화면과 보고 API가 빠르게 읽는 내부 SQLite 테이블입니다. 지금은 여기에 더클린커피 row를 저장하지 않았습니다.
4. Slack 보고 문구 실제 저장 전까지는 `Naver: 0원 확인 후보`와 `DB 캐시 저장 전` 경고를 같이 보여줍니다.

단계별 추진계획

  1. 더클린커피 광고비를 먼저 닫는다. 무엇을 하는가: 네이버 광고비 수집 스크립트에 더클린커피 후보 캠페인 6개만 읽는 조건을 넣습니다. 왜 하는가: 조건 없이 저장하면 바이오컴 광고비가 더클린커피 광고비처럼 섞일 수 있기 때문입니다. 어떻게 하는가: DB 저장 없이 미리보기 실행(preview: 저장하지 않고 결과만 보는 실행)을 돌려 후보 6개만 남는지 확인합니다.
  2. Slack 발송 전 미리보기를 주간·월간 정본으로 고정한다. 무엇을 하는가: Slack 발송 전 미리보기 문구를 주간/월간 기준으로 고정합니다. 왜 하는가: 실제 발송 전에 숫자와 경고 문구가 맞는지 확인해야 하기 때문입니다. 어떻게 하는가: 매출, 광고비, 매출 대비 광고비, 데이터 출처 경고, 보류 플랫폼을 한 메시지에 넣습니다.
  3. 제품별 매출을 붙인다. 무엇을 하는가: 채널별 총매출 옆에 잘 팔린 제품 TOP3를 붙입니다. 왜 하는가: 매출이 늘거나 줄었을 때 어느 상품이 원인인지 바로 알아야 광고비와 재고 판단을 할 수 있기 때문입니다. 어떻게 하는가: 스마트스토어는 PlayAuto(여러 판매 채널 주문을 한 운영DB 테이블에 모아 둔 수집 원천) 상품명, 쿠팡은 TeamKeto ordersheets(쿠팡 판매자 주문서 API, 상품명·수량·금액을 읽는 공식 통로), 자사몰은 Imweb order items(아임웹 주문 안의 상품별 행)를 봅니다.
  4. 바이오컴을 같은 구조로 확장한다. 무엇을 하는가: 더클린커피에서 만든 보고 구조를 바이오컴에도 적용합니다. 왜 하는가: 두 브랜드를 같은 방식으로 봐야 광고비 배분 판단이 가능하기 때문입니다. 어떻게 하는가: 바이오컴의 자사몰, NPay, Google, Meta, Naver 데이터 출처를 별도로 정리합니다.
  5. 실제 Slack 발송을 승인받아 운영에 올린다. 무엇을 하는가: 승인 후 정해진 Slack 채널에 자동 보고를 보냅니다. 왜 하는가: 매주 사람이 같은 조회를 반복하지 않게 하기 위해서입니다. 어떻게 하는가: 발송 채널, 주기, 실패 알림, 로그 보관 위치를 정한 뒤 켭니다. 광고 플랫폼 전환 전송과는 무관합니다.

제품별 매출은 이렇게 붙입니다

채널별 매출 보고가 “어디에서 팔렸는지”를 보여준다면, 제품별 매출은 “무엇이 팔렸는지”를 보여줍니다. Slack 메시지에는 자사몰, 스마트스토어, 쿠팡 총매출 아래에 각 채널의 TOP 상품 3개를 붙입니다.

자사몰 Imweb order items(아임웹 주문 안의 상품별 행)를 주문 총액과 맞춰 봅니다. NPay와 Toss가 같은 주문을 두 번 세지 않도록 먼저 주문 단위로 중복을 제거하고, 상품 라인이 최신이 아니면 TOP 상품은 참고값으로만 표시합니다.
스마트스토어 운영DB `tb_playauto_orders shop_name='스마트스토어'`에서 상품명, 옵션명, 수량, 금액을 읽습니다. PlayAuto는 여러 판매 채널 주문을 한 테이블에 모아 둔 수집 원천입니다.
쿠팡 TeamKeto ordersheets(쿠팡 판매자 주문서 API)에서 상품명, 수량, 금액만 읽습니다. TeamKeto 계정에는 커피와 팀키토 상품이 섞여 있으므로 coffee 상품은 더클린커피 매출로, teamketo 상품은 참고값으로 나눠 보여줍니다.

보고서가 지키는 안전선

  • NPay 클릭을 구매완료로 세지 않는다.
  • 광고 플랫폼 주장 매출을 내부 매출 합계에 더하지 않는다.
  • Naver `convAmt`는 네이버 주장값이므로 내부 매출이 아니다.
  • 운영DB 쓰기 없이 읽기와 미리보기만 한다.
  • Slack 실제 발송 전에는 발송 전 미리보기로 문구와 숫자를 확인한다.
  • Google/GA4/Meta/TikTok/Naver 전환 전송은 하지 않는다.
  • 캠페인 ID, 클릭 ID, 주문/결제/회원 식별자를 외부 보고에 노출하지 않는다.
  • VM Cloud는 보조 원장이고 운영DB와 구분한다.

다음에 가장 중요한 일

다음 한 단계는 네이버 광고비 수집 스크립트에 “더클린커피 캠페인 6개만 읽는 조건”을 넣고, DB 저장 없이 다시 미리보기로 확인하는 것입니다. 이 작업은 API를 바꾸는 일이 아니라 내부 수집 스크립트를 고치는 일입니다. 이 작업이 끝나면 더클린커피 광고비 출처가 `Meta 포함 + Naver 0원 후보 + Google/TikTok 보류`로 안정화됩니다.

그 다음은 Slack 발송 전 미리보기를 승인 가능한 최종 문구로 만들고, 바이오컴 보고서에도 같은 구조를 적용하는 것입니다.
