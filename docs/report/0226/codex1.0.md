## 조사 A: AI 에이전트 활동 상태 (AI Insights 패널)

Tab 0 오버뷰에 "AI 에이전트 활동 상태" 섹션이 있다.
긴급/스케일/추세/스케줄 4가지 우선순위로 인사이트를 보여준다.

### 조사 항목

1. **데이터 소스**: 이 인사이트 텍스트는 어디서 오는가?
   - 백엔드 API에서 생성하는가? → 어떤 API 엔드포인트인가?
   - 프론트에서 계산하는가? → 어떤 데이터를 기반으로?
   - LLM(OpenAI/Claude API)으로 생성하는가? → 프롬프트는 어디에 있는가?
   - 하드코딩/정적 텍스트인가?

2. **생성 로직**: 인사이트의 우선순위(긴급/스케일/추세/스케줄)는 어떻게 결정되는가?
   - 규칙 기반인가? (예: CTR이 X% 이하이면 "긴급")
   - AI 기반인가? (LLM이 판단)
   - 수동 설정인가?

3. **업데이트 주기**: 인사이트는 언제 갱신되는가?
   - 페이지 로드마다?
   - 백엔드에서 주기적으로?
   - 수동 갱신?

4. **관련 파일 목록**: 이 기능과 관련된 파일들을 모두 나열해라.
   - 프론트: 컴포넌트 파일, 타입 정의
   - 백엔드: API 엔드포인트, 생성 로직
   - 프롬프트 파일 (있다면)

5. **현재 인사이트 예시**: 현재 코드에서 생성되는 인사이트 텍스트 예시를 3-5개 뽑아라.

### 리포트 형식

```
## AI 에이전트 활동 상태 구조 분석

### 데이터 흐름
[소스] → [처리] → [프론트 표시]

### 관련 파일
- 프론트: ...
- 백엔드: ...
- 기타: ...

### 생성 방식
[규칙 기반 / AI 기반 / 정적 / 혼합] — 상세 설명

### 우선순위 결정 로직
...

### 현재 인사이트 예시
1. [긴급] "..."
2. [스케일] "..."
...

### 개선 가능 포인트 (의견)
- ...
```

## 조사 B: 키워드 인텐트 분석

Tab 0 오버뷰 하단에 "키워드 인텐트 분석" 섹션이 있다.
정보성/상업성/탐색성/브랜드 비율을 바 차트로 보여준다.

### 조사 항목

1. **데이터 소스**: 인텐트 분류는 어디서 오는가?
   - GSC 키워드 목록을 가져와서 분류하는가?
   - 백엔드에서 분류하는가? → 분류 로직은?
   - LLM으로 분류하는가?
   - 프론트에서 규칙 기반으로 분류하는가?

2. **분류 기준**: "정보성", "상업성", "탐색성", "브랜드"의 정확한 분류 기준은?
   - 키워드에 특정 단어가 포함되면 분류? (예: "가격" → 상업성)
   - LLM이 판단?
   - 외부 도구 연동?

3. **대표 키워드 선정**: 각 인텐트별 대표 키워드 예시는 어떻게 선정되는가?

4. **업데이트 주기**: GSC 데이터 기준 기간은? 갱신 주기는?

5. **관련 파일 목록**: 이 기능 관련 파일 모두 나열.

6. **Tab 2(키워드 분석)와의 관계**: 
   Tab 2의 키워드 순위 추적 데이터와 Tab 0의 인텐트 분석이 같은 데이터를 쓰는가?
   별개인가?

### 리포트 형식

```
## 키워드 인텐트 분석 구조

### 데이터 흐름
[GSC 데이터] → [분류 로직] → [프론트 표시]

### 분류 로직 상세
- 정보성: [기준]
- 상업성: [기준]
- 탐색성: [기준]
- 브랜드: [기준]

### 관련 파일
- ...

### Tab 2 키워드 분석과의 관계
- 공유 데이터: ...
- 별개 데이터: ...

### 개선 가능 포인트 (의견)
- ...
```
