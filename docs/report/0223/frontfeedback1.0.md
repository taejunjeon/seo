너는 Biocom AI SEO/AEO/GEO 대시보드의 시니어 프론트엔드 엔지니어다(Next.js App Router).

목표: “AI 유입(AI 서비스 referral 유입)”을 오버뷰 + 사용자 행동 탭에서 의사결정용으로 보여준다.
전제: 백엔드에 /api/ga4/ai-traffic(또는 유사 엔드포인트)가 생긴다.

[1] 오버뷰(요약 카드) 추가
- 카드 1개로 요약:
  AI 유입 세션 / AI 활성 사용자 / AI 구매(또는 key event) / AI 매출
- 주석(중요): “AI 유입=ChatGPT/Perplexity/Gemini/Claude 등 AI 서비스에서 직접 넘어온 유입. Google 검색의 AI Overview 유입은 여기 포함되지 않을 수 있음.”

[2] 사용자 행동 탭에 “AI 유입” 섹션 추가
- 기간 선택(7/30/90/커스텀)이 이미 있으면 재사용.
- 테이블 2개:
  (A) 소스별(sessionSource/sessionSourceMedium) 세션/사용자/구매/매출
  (B) 랜딩페이지별(landingPagePlusQueryString) 세션/사용자/구매/매출
- 랜딩페이지 row 클릭 시:
  “페이지 진단(Tab 6)”으로 이동 + URL 자동 입력 + (정밀 진단 버튼까지 원클릭이면 베스트)

[3] “AI 유입 주제(근사)” UI(옵션)
- 랜딩페이지 TOP 클릭하면 하단에 “이 페이지의 GSC 상위 쿼리 TOP 3” 펼쳐보기(백엔드가 제공하면)
- 이걸 ‘AI 프롬프트’라고 부르지 말고 “AI 유입이 많이 landing된 페이지의 검색 주제(근사)”라고 표기.

[4] UX
- 로딩: 0.5초 이상이면 skeleton + “캐시된 결과일 수 있음(업데이트 시간 표시)” 문구.
- 0 데이터일 때도 빈 화면 금지: “최근 기간에 AI referral 유입이 0으로 측정됨” 안내 + 측정 정의 표시.

완료 기준(DoD):
- 오버뷰에서 AI 유입 KPI가 한눈에 보임
- 사용자 행동 탭에서 소스/랜딩 2개 테이블이 보임
- 랜딩 클릭 -> 페이지 진단으로 자연스럽게 이어짐
- “AI 유입 정의”를 사용자 혼동 없이 명확히 보여줌
