> **현재 상태**: Phase 2 완료 기준

## 메뉴 구조 개요

CRM 대시보드는 **상단 KPI 카드 3장 + 기능 카드 4장** 으로 진입하며, 내부에 4개 탭으로 구성된다.

### 상단 KPI 카드 (공통 요약)

| KPI | 현재 값(예시) | 설명 |
|-----|-------------|------|
| 완료 상담 | 3,947건 (고유 고객 3,630명) | 전체 상담 완료 건수와 고유 고객 수 |
| 90일 전환율 | 26.1% (전환 730명 / 성숙 2,796명) | 상담 완료 후 90일 내 구매 전환한 고객 비율 |
| 상담 효과 추정 매출 | 2.3억원 | 상담 고객 vs 미상담 고객의 매출 차이에서 추정한 증분 매출 |

## 기능 카드 4장 (탭 진입점)

### 1. 상담사 가치 분석 (`/callprice`)

**아이콘**: 차트 (bar chart)
**설명**: 상담사별 성과, 상담 효과 추정, 충원 시나리오 시뮬레이션

#### 대표(TJ)가 보는 핵심 화면
- **상담 1건당 추정 가치**: 상담사를 추가 고용했을 때 기대 수익을 계산하는 근거
- **충원 시나리오 시뮬레이션**: "상담사 1명 추가하면 월 매출 얼마 증가?" 시뮬레이션
- **구독 vs 상담 전환 비교**: 구독 상품과 상담 효과의 ROI 비교

#### 상담 팀장이 보는 핵심 화면
- **상담사별 성과 테이블**: 상담 건수, 완료율, 전환율, 건당 가치 비교
- **분석유형별 상담 가치**: 음식물과민증/유기산/미네랄 등 검사 종류별 상담 효과
- **평일 vs 주말 비교**: 요일별 상담 효과 차이 → 근무 시간 최적화 근거
- **수습 가이드**: 신규 상담사의 초기 성과 기대치와 기준선

#### 운영팀이 보는 핵심 화면
- **영양제 구매 타이밍**: 상담 후 영양제 첫 구매까지 걸리는 일수 분포 → 후속 알림 타이밍 설정
- **영양제 재구매 패턴**: 재구매 주기와 패턴 → 리텐션 전략 수립

---

### 2. 코호트 / 북극성 지표 (CRM 탭 내 + `/callprice`)

**아이콘**: 상승 차트
**설명**: 성숙 기간별 전환율/매출 비교, 90일 재구매 순이익 추적

#### 대표(TJ)가 보는 핵심 화면
- **90일 재구매 순이익**: 회사의 건강 상태를 하나의 숫자로 추적하는 북극성 지표
- **월별 코호트 히트맵**: 첫 구매 월 기준으로 M+1, M+2, M+3... 재구매율 추적
- **성숙 기간별 전환율**: 30일/60일/90일 윈도우에서 전환율이 어떻게 변하는지

#### 마케터가 보는 핵심 화면
- **검사 유형별 코호트**: 음식물과민증 구매 코호트 vs 미네랄 구매 코호트의 재구매 행동 차이
- **결제채널별 분해**: 어떤 채널에서 온 고객이 더 오래 남는지

### 3. CRM 관리 허브 (`/crm` 후속 관리 탭)

**아이콘**: 타겟/과녁
**설명**: 후속 관리 대상, 실험 운영, 결제 귀속 진단

이 카드가 `/crm` 페이지의 메인 진입점이며, 내부에 4개 탭을 포함한다.

#### 탭 1: 후속 관리

**누가, 언제, 왜 쓰는가**:
- **상담 팀장** — 매일 아침, "오늘 후속 연락할 고객이 누구인지" 확인
- **운영팀** — 주 1회, "상담 완료 후 아직 구매하지 않은 고객"을 골라 후속 액션 실행

**주요 기능**:

| 기능 | 설명 | 페르소나 |
|------|------|----------|
| 시나리오 선택 | "상담 완료 → 미구매 후속" / "부재/변경 → 재연락 대상" 2가지 | 운영팀 |
| 후속 대상 리스트 | 고객명, 연락처, 상담사, 검사유형, 상담일, 상태, 주문 여부, 추천 액션 | 상담 팀장 |
| 추천 액션 배지 | 구매 유도 / 영양제 추천 / 재연락 / 후속 확인 — 데이터 기반 자동 분류 | 운영팀 |
| 상담사별 요약 | 상담사별 건수, 완료율, 주문 매칭률 KPI | 대표 |
| 주문 매칭 현황 | 상담-주문 매칭 통계 (전화번호 기반) | 운영팀 |

#### 탭 2: 실험 운영

**누가, 언제, 왜 쓰는가**:
- **대표(TJ)** — "이 CRM 캠페인이 실제로 추가 매출을 만들었는가?" 확인
- **마케터** — A/B 실험 생성, treatment/control 비교, 전환 동기화

**주요 기능**:

| 기능 | 설명 | 페르소나 |
|------|------|----------|
| 실험 생성 | 실험명, 채널(채널톡/알리고/수동), 전환 윈도우, 가설 입력 | 마케터 |
| 실험 목록 | 상태별(active/completed/draft) 실험 카드 | 모든 사용자 |
| 전환 동기화 | 운영 DB 주문과 실험 배정을 자동 매칭 → 버튼 1클릭 | 운영팀 |
| Variant 비교 | treatment vs control 구매율/순매출 비교 차트 | 대표 |
| Variant 성과 테이블 | 배정 수, 구매자 수, 순매출, 구매율 상세 | 마케터 |
| 최근 배정 로그 | 고객별 variant 배정 이력 | 운영팀 |

#### 탭 3: 알림톡 발송

**현재 상태**: 준비 단계 (환경변수 미설정)

**누가, 언제, 왜 쓰는가**:
- **운영팀** — 후속 관리 대상에게 카카오 알림톡 발송
- **마케터** — 실험에서 treatment 그룹에게 메시지 발송

**주요 기능** (완성 시):

| 기능 | 설명 | 상태 |
|------|------|------|
| 연동 상태 확인 | Sender Key/API Key/User ID/Phone 설정 여부 | UI 완성, 설정 미완 |
| 대상 선택 | 후속 관리 대상 리스트에서 발송 대상 선택 | 미구현 |
| 템플릿 선택 | 카카오 승인 템플릿 선택 | 미구현 |
| 발송 | 알리고 API 통해 알림톡 발송 | 미구현 |
| 결과 확인 | 발송 성공/실패/열람 현황 | 미구현 |

**필요 환경변수**: `ALIGO_API_KEY`, `ALIGO_USER_ID`, `ALIGO_SENDER_PHONE`

#### 탭 4: 결제 귀속

**누가, 언제, 왜 쓰는가**:
- **대표(TJ)** — "GA4에서 (not set)으로 잡히는 매출의 실제 출처가 뭔지" 진단
- **개발팀** — PG 결제 → GA4 전환 추적의 정합성 확인

**주요 기능**:

| 기능 | 설명 | 페르소나 |
|------|------|----------|
| 원장 요약 카드 | live/replay/smoke 결제 이벤트 분포 | 개발팀 |
| 토스 조인율 | 토스 승인 건과 결제 이벤트의 매칭률 | 대표 |
| GA4 (not set) 비율 | 출처 불명 매출 비율과 금액 | 대표 |
| 날짜별 추이 차트 | GA4 (not set) 구매 vs 토스 승인 vs 결제 이벤트 일별 비교 | 개발팀 |
| 날짜별 비교표 | 10개 열의 일자별 상세 진단 테이블 | 개발팀 |

### 4. 더클린커피 전략 (계획 단계)

**아이콘**: 커피 컵

**현재 상태**: 기획 단계 — 구체적인 UI 미구현

**향후 기능 방향**:
- 재구매율/LTR(Lifetime Revenue) 코호트 분석
- 가격 민감도 시뮬레이션
- CRM 메시지 전략 (윈백/업셀/크로스셀) 설계 도구
