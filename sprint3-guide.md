## 3.1 DataTable → Tab 1 / Tab 2 적용

### 지금 상태 (Before)

대시보드에는 데이터를 표(table)로 보여주는 탭이 여러 개 있소.

- **Tab 1** (칼럼 분석): 칼럼별 클릭수, 노출수, CTR 등을 보여주는 표
- **Tab 2** (키워드 분석): 키워드별 순위, 클릭수 등을 보여주는 표

이 두 탭은 HTML `<table>` 태그를 직접 써서 표를 그리고 있소. 문제는:

1. **정렬이 안 됨** — 클릭수 순으로 보고 싶어도 정렬 버튼이 없소
2. **검색이 안 됨** — 키워드가 100개이면 스크롤해서 찾아야 하오
3. **페이지네이션이 없음** — 데이터가 많으면 끝없이 스크롤해야 하오

그런데 이미 프로젝트에 `DataTable`이라는 컴포넌트가 만들어져 있소 (`/frontend/src/components/common/DataTable.tsx`). 이 컴포넌트는 **정렬, 검색, 페이지네이션**을 모두 지원하오. 다른 탭에서는 이미 쓰고 있소.

### 할 일 (What)

Tab 1과 Tab 2의 수동 `<table>` 코드를 **DataTable 컴포넌트**로 교체하오.

### 왜? (Why)

| 비유 | 설명 |
|------|------|
| 학교 급식 | 반마다 따로 밥을 차리는 대신, 급식실 한 곳에서 통일해서 배식하면 효율적이오 |
| 실제 효과 | 정렬/검색/페이지네이션이 자동으로 생기고, 코드도 짧아지오 |

**사용자 입장**: "클릭수가 가장 높은 키워드가 뭐지?" → 헤더 클릭 한 번으로 정렬 가능
**개발자 입장**: 표 관련 버그를 DataTable 한 곳에서만 고치면 전체에 반영됨

### 예시

```
[Before] Tab 2 키워드 표
┌──────────────┬───────┬───────┬──────┐
│ 키워드        │ 클릭수 │ 노출수  │ 순위  │  ← 정렬 안 됨, 검색 안 됨
├──────────────┼───────┼───────┼──────┤
│ 유전자 검사    │   45  │  1200  │  8.3 │
│ DNA 분석      │   32  │   890  │ 12.1 │
│ ...100개 더...│       │        │      │  ← 스크롤 지옥
└──────────────┴───────┴───────┴──────┘

[After] DataTable 적용 후
🔍 검색: [유전자________]          페이지: ◀ 1 2 3 ▶
┌──────────────┬───────▼┬───────┬──────┐
│ 키워드        │ 클릭수 ▼│ 노출수  │ 순위  │  ← 헤더 클릭으로 정렬
├──────────────┼────────┼───────┼──────┤
│ 유전자 검사    │   45   │  1200  │  8.3 │
│ 유전자 치료    │   28   │   650  │ 15.2 │
└──────────────┴────────┴───────┴──────┘
```

## 3.2 AEO Score 분포 바 (Tab 1)

### 지금 상태 (Before)

**AEO Score**란 "AI Engine Optimization Score"의 약자로, 특정 페이지가 AI 검색엔진(ChatGPT, Perplexity 등)에 얼마나 잘 인용되는지를 0~100점으로 나타낸 점수이오.

현재 Tab 1(칼럼 분석)에서 AEO Score는 표 안의 숫자로만 보이오. 예를 들어 "72점"이라고만 적혀 있을 뿐, **그게 높은 건지 낮은 건지 직관적으로 알기 어렵소.**

### 할 일 (What)

AEO Score 옆에 **색상 막대(분포 바)**를 추가해서, 점수가 시각적으로 보이게 하오.

### 왜? (Why)

| 비유 | 설명 |
|------|------|
| 시험 성적표 | 점수가 "72점"이라고만 쓰여 있으면 모르지만, 옆에 빨강-노랑-초록 막대가 있으면 한눈에 파악 가능 |
| 실제 효과 | 어떤 페이지의 AI 최적화가 부족한지 **1초 만에** 알 수 있소 |

### 예시

```
[Before] 숫자만 표시
│ AEO Score │
│    72     │  ← 이게 좋은 건지 나쁜 건지?
│    31     │
│    89     │

[After] 분포 바 추가
│ AEO Score                    │
│ 72  ████████████████░░░░░░░  │  ← 노란색 (보통)
│ 31  ████████░░░░░░░░░░░░░░░  │  ← 빨간색 (낮음)
│ 89  ██████████████████████░  │  ← 초록색 (좋음)
```

점수 구간별 색상:
- **0~40**: 빨간색 — AI 검색에서 거의 인용 안 됨
- **41~70**: 노란색 — 보통, 개선 여지 있음
- **71~100**: 초록색 — AI 검색 최적화가 잘 되어 있음

## 3.3 소스별 CSS 색상 (AI Traffic)

### 지금 상태 (Before)

"AI Traffic" 섹션은 ChatGPT, Perplexity, Google AI Overview 등 AI 검색엔진에서 우리 사이트로 들어온 방문자를 보여주는 곳이오.

현재 문제: **모든 AI 소스가 같은 색상(남색)**으로 표시되오. ChatGPT에서 온 트래픽이든 Perplexity에서 온 트래픽이든 다 같은 색깔의 "AI" 뱃지가 붙어 있소.

### 할 일 (What)

각 AI 소스마다 **고유한 브랜드 색상**을 적용하오.

| AI 소스 | 브랜드 색상 | 색상 코드 |
|---------|-----------|----------|
| ChatGPT | 초록색 | #10A37F |
| Perplexity | 보라색 | #5A67D8 |
| Google AI Overview | 파란색 | #4285F4 |
| Claude | 주황색 | #D97706 |
| Copilot | 하늘색 | #0EA5E9 |

### 왜? (Why)

| 비유 | 설명 |
|------|------|
| 지하철 노선도 | 모든 노선이 같은 색이면? 2호선인지 3호선인지 구분 불가. 각 노선마다 고유 색상이 있어야 한눈에 파악 가능 |
| 실제 효과 | "ChatGPT에서 유입이 많구나" vs "Perplexity에서 유입이 많구나"를 **색상만 보고** 바로 구분 가능 |

### 예시

```
[Before] 전부 같은 색
  AI  ChatGPT     ████████  120회
  AI  Perplexity  ████       45회    ← 전부 남색, 구분 어려움
  AI  Google AIO  ██████     88회

[After] 소스별 고유 색상
  🟢  ChatGPT     ████████  120회   ← 초록색
  🟣  Perplexity  ████       45회   ← 보라색
  🔵  Google AIO  ██████     88회   ← 파란색
```

## 3.4 KPI 카드 디자인 통일

### 지금 상태 (Before)

대시보드 곳곳에 **KPI 카드**가 있소. KPI(Key Performance Indicator)란 "핵심 성과 지표"로, 중요한 숫자를 카드 형태로 보여주는 UI 요소이오.

예를 들어:
- 개요 탭: "총 클릭수 1,234" 카드
- CWV 탭: "LCP 2.1초" 카드
- 퍼널: "전체 전환율 3.2%" 카드

**문제**: 같은 "카드"인데 탭마다 디자인이 미묘하게 다르오.

| 속성 | 개요 탭 KPI 카드 | CWV 탭 상세 카드 |
|------|:---------------:|:--------------:|
| 모서리 둥글기 | 16px | 12px |
| 안쪽 여백 | 20px | 18px |
| 그림자 | 있음 | 약간 다름 |

4px 차이가 뭐가 중요하냐고? 이건 마치 **같은 학교 교복인데 반마다 단추 위치가 2mm씩 다른 것**과 같소. 하나하나는 안 보이지만, 전체적으로 보면 **"뭔가 어설퍼 보이는"** 느낌을 주오.

### 할 일 (What)

모든 KPI 카드의 CSS를 하나의 표준으로 통일하오.

```
표준 KPI 카드:
  border-radius: 16px      ← 모서리 둥글기 통일
  padding: 20px             ← 안쪽 여백 통일
  box-shadow: 통일된 그림자    ← 그림자 통일
  font-size: 기준값 적용      ← 글자 크기 통일
```

### 왜? (Why)

| 비유 | 설명 |
|------|------|
| 애플 제품 디자인 | 아이폰, 아이패드, 맥북 — 다른 제품이지만 모서리 둥글기, 색상 톤이 통일되어 있어서 "애플답다"는 느낌을 줌 |
| 실제 효과 | 대시보드 전체가 **하나의 디자인 시스템**으로 보이게 되어 전문적이고 신뢰감 있는 인상을 줌 |

### 예시

```
[Before] 탭마다 카드 디자인이 제각각

  ┌─────────────┐  ╭─────────────╮   ← 모서리 다름
  │  클릭수       │  │  LCP         │
  │  1,234      │  │  2.1초       │   ← 여백 다름
  │             │  │             │
  └─────────────┘  ╰─────────────╯

[After] 모든 카드가 같은 규격

  ╭─────────────╮  ╭─────────────╮   ← 모서리 통일
  │  클릭수       │  │  LCP         │
  │  1,234      │  │  2.1초       │   ← 여백 통일
  │             │  │             │
  ╰─────────────╯  ╰─────────────╯
```

## 정리: 4개 작업 한눈에 보기

| # | 작업 | 한마디 요약 | 난이도 |
|:-:|------|-----------|:-----:|
| 3.1 | DataTable 적용 | 표에 정렬/검색/페이지넘김 추가 | 중 |
| 3.2 | AEO Score 분포 바 | 숫자를 색상 막대로 시각화 | 소 |
| 3.3 | AI Traffic 소스 색상 | AI 소스마다 고유 색상 부여 | 소 |

### 공통점

이 4개 작업은 모두 **"기능은 이미 있지만, 더 보기 좋고 쓰기 편하게"** 만드는 작업이오.
새로운 API를 만들거나, 데이터를 바꾸는 작업은 하나도 없소.
순수하게 **프론트엔드(화면) 개선** 작업이오.

### 왜 Sprint 3에서 하는가?

```
Sprint 1: 코드 정리 (4,000줄짜리 파일을 나눔)     ← 완료
Sprint 2: 가짜 데이터 제거 (실데이터만 사용)        ← 완료
Sprint 3: UI 마무리 (보기 좋게, 쓰기 편하게)       ← 지금 여기
Sprint 4: 서버 보안/안정성                        ← 다음
```

Sprint 1~2에서 **구조와 데이터**를 정리했으니, 이제 **겉모습**을 다듬는 순서이오.
집으로 비유하면: 골조(Sprint 1) → 배관/전기(Sprint 2) → **인테리어(Sprint 3)** → 보안시스템(Sprint 4)이오.
