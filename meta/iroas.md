## 2. iROAS란 무엇이고, 왜 측정해야 하는가

### ROAS vs iROAS

```
ROAS = 광고 귀속 매출 / 광고비
       "광고를 클릭한 사람이 산 금액"

iROAS = 증분 매출 / 광고비
        "광고가 없었다면 발생하지 않았을 매출"
```

**왜 다른가?**: 광고를 클릭하고 구매한 사람 중 일부는 광고가 없어도 검색이나 직접 방문으로 구매했을 것이다. ROAS는 이런 "자연 전환"까지 광고 공으로 잡지만, iROAS는 **진짜 광고 덕분에 생긴 매출만** 계산한다.

### 바이오컴에서 특히 중요한 이유

바이오컴 검사키트는 **브랜드 검색 → 비교 → 구매**의 고관여 구매 경로를 가진다. 고객이 "바이오컴 유전자검사"를 검색해서 들어왔는데, 그 전에 Meta 광고를 본 적이 있으면 Meta가 그 전환을 자기 것으로 잡는다. 이런 경우 ROAS는 높지만 iROAS는 낮을 수 있다.

## 3. iROAS 측정 방법

### Claude Code 의견

**권장: 2단계 접근법**

#### 1단계: 캠페인별 7d_click vs 1d_click 비율로 "증분 추정치" 산출 (즉시 가능)

현재 데이터만으로도 증분 비율을 추정할 수 있다:

```
1d_click 전환 = 839건 → "광고 보자마자 산 사람" (증분 가능성 높음)
7d_click 전환 = 987건 → "클릭 후 1주 내 산 사람"
기본 전환     = 1,292건 → "조회만 한 사람 포함" (증분 가능성 낮음)
```

**증분 추정 로직:**
- 1d_click 전환의 **80~90%**는 진짜 증분 (광고의 즉각 효과)
- 7d_click - 1d_click 차이(148건)의 **50~70%**는 증분 (고민 후 구매)
- 기본 - 7d_click 차이(305건)의 **10~30%**만 증분 (대부분 자연 전환)

이 비율을 적용하면:
- 증분 전환 추정: 839×0.85 + 148×0.6 + 305×0.2 = **713 + 89 + 61 = 약 860건**
- 증분 매출: 860 × ₩462,000 = **₩3.97억**
- **추정 iROAS: ₩3.97억 / ₩1.17억 = 3.40x**

#### 2단계: Meta Conversion Lift 실험으로 검증 (2~3주 소요)

1단계의 추정치를 실제 실험으로 확인:

1. Meta Business Suite > Experiments > Conversion Lift Study 생성
2. 전체 예산의 10~15%를 holdout(광고 안 보여주는 그룹)에 배정
3. 2~3주 후 holdout 그룹 vs 광고 노출 그룹의 구매율 차이 = 증분 전환율
4. 증분 전환율 × 광고 노출 고객 수 = 증분 전환 건수
5. 증분 전환 × AOV / 광고비 = **실제 iROAS**

**주의**: Meta가 자기 플랫폼을 측정하므로 결과가 약간 관대할 수 있음. 의심되면 Geo-Lift(지역별 광고 중단) 실험으로 교차 검증.

### Codex 의견

**Meta Conversion Lift가 가장 현실적이고, Geo-Lift가 차선책.**

- **Meta Conversion Lift (추천 1순위)**: Meta가 자체적으로 test/holdout 그룹을 나누어 증분 전환을 측정. 바이오컴 규모(월 1.17억)면 충분히 활용 가능. 일반적으로 월 5천만 원 이상 집행하면 통계적 유의성 확보 가능. 별도 인프라 불필요, 2~4주 내 결과 산출. 단, Meta가 자기 플랫폼을 측정하므로 이해충돌이 있고 결과가 약간 관대할 수 있음.

- **Geo-Lift (추천 2순위)**: 지역별로 광고를 켜고 끄는 방식. 바이오컴은 전국 배송이므로 가능. 한국 시장 특성상 지역 간 소비 패턴 차이가 크고, 충분한 표본 모으려면 4~8주 필요. Meta의 GeoLift 오픈소스 R 패키지 활용 가능.

- **Holdout Test (전체 중단)**: 가장 깨끗하지만 월 1.17억 규모에서 전체 중단은 매출 리스크가 큼. 부분 holdout(특정 캠페인이나 오디언스 세그먼트)이 현실적 타협안.

**iROAS 추정치**: 현재 ROAS 5.03x에서 실제 iROAS는 **1.5~2.5x 범위**로 추정. 7d_click 전환 987건 중 증분 비율 50~70% 가정 시 증분 전환 494~691건, 월 광고비 ₩1.17억 대비 iROAS 1.95~2.73x. "광고 없이도 샀을 사람"은 기본 기준 50~65%, 7d_click 기준 30~50%.

## 4. 광고 집행 판단 기준 — 언제 광고를 계속하고, 언제 멈추는가

### 손익분기 ROAS 계산

```
손익분기 ROAS = 1 / 마진율

마진율 40% → 손익분기 ROAS = 2.5x
마진율 50% → 손익분기 ROAS = 2.0x
마진율 60% → 손익분기 ROAS = 1.67x
```

현재 7d_click ROAS 2.38x:
- 마진율 50%면 → 첫 구매만으로 **수익** (2.38 > 2.0)
- 마진율 40%면 → 첫 구매만으로 **거의 손익분기** (2.38 ≈ 2.5)

### 재구매를 반영한 LTV 기반 판단

| 구분 | 마진율 40% | 마진율 50% | 마진율 60% |
|------|-----------|-----------|-----------|
| 첫 구매 마진 | ₩184,800 | ₩231,000 | ₩277,200 |
| 12개월 재구매 마진 (34.9%) | ₩64,495 | ₩80,619 | ₩96,743 |
| **12개월 LTV** | **₩249,295** | **₩311,619** | **₩373,943** |
| **LTV / CPA** | **2.65x** | **3.31x** | **3.98x** |

**현재 LTV/CPA 3.31x(마진 50% 가정)는 건강함.**

### Claude Code — 광고 집행 판단 기준

#### 중단/유지/확대 기준표

| 상황 | 7d_click ROAS | LTV/CPA | 판단 | 이유 |
|------|-------------|---------|------|------|
| **확대** | > 3.0x | > 3.0x | 예산 20%/주 증가 | 첫 구매에서 이미 수익 + LTV 보너스 |
| **유지** | 2.0~3.0x | > 2.5x | 현행 유지 + 최적화 | 첫 구매 수익 + LTV로 여유 |
| **경계** | 1.5~2.0x | 2.0~2.5x | 소재 교체 + 캠페인 정리 | 첫 구매 손실 가능, LTV로만 회수 |
| **축소** | 1.0~1.5x | < 2.0x | 예산 30% 축소, 상위 캠페인 집중 | LTV 반영해도 위험 |
| **중단** | < 1.0x | < 1.5x | 즉시 중단, 구조 재검토 | 돈을 태우고 있음 |

#### "첫 구매는 손실이지만 재구매로 보완" — 어디까지 버틸 수 있는가?

```
CPA가 AOV × 마진율보다 높으면 첫 구매는 적자.

예: CPA ₩94,000 vs 첫 구매 마진 ₩231,000 (마진율 50%)
→ 첫 구매에서 ₩137,000 수익. 여유 있음.

만약 CPA가 ₩250,000으로 올라가면?
→ 첫 구매 마진 ₩231,000 - CPA ₩250,000 = -₩19,000 적자
→ 12개월 재구매 마진 ₩80,619로 보완 → 순이익 ₩61,619
→ 아직 수익이므로 광고 유지 가능

CPA 한계선: 첫 구매 마진 + 12개월 재구매 마진 = ₩311,619
→ CPA ₩311,619 이상이면 LTV 반영해도 적자 → 이때 멈춰야 함
```

**현실적 CPA 한계:**
- 마진율 50% 기준: **CPA ₩250,000까지 버틸 수 있음** (현재 ₩94,000이므로 여유 큼)
- 마진율 40% 기준: **CPA ₩200,000까지** 

#### 영업이익 25억 목표 관점

```
현재:
- 월 광고비: ₩1.17억 × 12 = 연 ₩14억
- 7d_click 광고 매출: ₩14억 × 2.38 = ₩33.3억
- 광고 기여 매출이익: ₩33.3억 × 50%(마진) = ₩16.65억
- 광고 기여 영업이익: ₩16.65억 - ₩14억(광고비) = ₩2.65억

→ 광고가 영업이익 25억 중 ₩2.65억(10.6%)을 만들고 있음

만약 광고비를 ₩1.5억/월로 올리면?
- 연 ₩18억 광고비
- ROAS가 2.0x로 소폭 하락 가정 → 매출 ₩36억
- 매출이익 ₩18억 - 광고비 ₩18억 = 영업이익 기여 ₩0
→ ROAS 하락 없이 확대할 수 있는지가 핵심

결론: 광고비 확대는 가능하지만, 7d_click ROAS가 2.0x 아래로 떨어지면
      영업이익 기여가 0 이하가 되므로 그 전에 멈춰야 함
```

### Codex — 광고 집행 판단 기준

#### 중단 기준

- **즉시 중단**: 7d_click ROAS < 1.5x (마진율 40% 기준, LTV 반영해도 회수 불가)
- **경고 구간**: 7d_click ROAS 1.5~2.0x (LTV로만 회수 가능, 캐시플로우 부담)
- **건강 구간**: 7d_click ROAS > 2.5x

#### 캠페인별 차별 기준

개별 캠페인 중단 기준은 3가지를 모두 충족할 때:
1. 7d_click ROAS < 손익분기 ROAS(2.0~2.5x)가 **2주 연속** 지속
2. **그리고** 해당 캠페인의 재구매율이 평균(26.3%) 이하
3. **그리고** 상담 연결율이 평균 이하

ROAS만으로 자르면 LTV가 높은 캠페인을 놓칠 수 있음.

#### 예산 확대 기준

7d_click ROAS > 5x이고 일예산 대비 노출이 포화되지 않았으면, 주 20% 단위로 예산 증가. 음식물 과민증 캠페인(ROAS 13.21x)이 여기에 해당.

#### 월 1.17억 적정성

현재 규모에서는 광고비를 늘려도 됨. **한계 ROAS(마지막 추가 1원의 수익률)가 떨어지는 지점**을 모니터링해야 함. 주간 단위로 광고비를 10~20% 올려보면서 7d_click ROAS가 2.0x 아래로 떨어지지 않는 한 확대 가능.

#### 상담 효과가 핵심 레버

광고 획득 고객의 상담 연결율 향상이 LTV에 가장 큰 영향. 상담 연결율을 10%p 올리면 CPA 허용 한계가 ₩8,284/명 올라감. 광고비 최적화보다 **"광고로 데려온 고객을 상담으로 연결하는 비율"**을 올리는 게 영업이익에 더 큰 영향을 줄 수 있음.

## 6. "첫 구매 적자여도 광고를 계속할 수 있는가?" — 시뮬레이션

### 시나리오 A: 현재 상태 (CPA ₩94K, 마진율 50%)

```
첫 구매: AOV ₩462,000 × 50% = 마진 ₩231,000 - CPA ₩94,000 = +₩137,000 ✅
재구매 (12개월): ₩462,000 × 34.9% × 50% = +₩80,619
12개월 순이익/고객: ₩137,000 + ₩80,619 = +₩217,619

→ 첫 구매에서 이미 수익. 매우 건강.
```

### 시나리오 B: CPA가 2배로 올라간 경우 (CPA ₩200K)

```
첫 구매: 마진 ₩231,000 - CPA ₩200,000 = +₩31,000 (겨우 수익)
재구매: +₩80,619
12개월 순이익/고객: +₩111,619

→ 아직 수익. 광고 유지 가능. 단 캐시플로우 압박 있음.
```

### 시나리오 C: 첫 구매 적자 (CPA ₩280K)

```
첫 구매: 마진 ₩231,000 - CPA ₩280,000 = -₩49,000 ❌
재구매: +₩80,619
12개월 순이익/고객: +₩31,619

→ 첫 구매 적자이지만, 재구매로 보완됨. 12개월 기다리면 수익.
→ 영업이익 25억 목표 관점: 영업이익이 양수이므로 광고 유지 가능.
→ 단, 캐시플로우 관리 필수 (고객 확보 후 3~6개월은 적자).
```

### 시나리오 D: 광고 중단 필요 (CPA ₩350K)

```
첫 구매: 마진 ₩231,000 - CPA ₩350,000 = -₩119,000 ❌
재구매: +₩80,619
12개월 순이익/고객: -₩38,381 ❌

→ 재구매 반영해도 적자. 광고 중단 필요.
```

### Claude Code 결론

| CPA | 첫 구매 | 12개월 LTV | 판단 |
|-----|---------|----------|------|
| < ₩200K | +수익 | +수익 | **적극 확대** |
| ₩200~250K | +겨우 수익 | +수익 | **유지 + 최적화** |
| ₩250~310K | -적자 | +수익 | **유지 가능** (영업이익 목표 관점. 재구매 회수에 3~6개월) |
| > ₩310K | -적자 | -적자 | **중단** |

### Codex 결론

LTV/CPA > 3.0x이면 건강, 2.0~3.0x이면 최적화 필요, 2.0x 미만이면 위험. 현재 3.31x로 건강한 수준이나, **이것은 상담 효과를 반영하지 않은 수치**임. 상담 연결 고객의 LTV는 비상담의 8.4배(₩94,078 vs ₩11,238)이므로, 광고 → 상담 퍼널이 작동하면 CPA 한계가 크게 올라감.

## 8. 캠페인별 재구매율 · 상담연결률 · 환불률 — 산출 가능 여부 분석

### Claude Code 분석

#### 현재 데이터 연결 상태

```
Meta 캠페인 (9개)
  ↓ campaign_name, 전환 건수/매출
  ✕ 전환 고객의 개인정보(전화번호/이메일)는 Meta가 안 줌
  
Attribution Ledger (235건)
  ↓ utm_source, utm_campaign, orderId, fbclid
  ↓ meta 계열 UTM: 26건만 (전체의 11%)
  ↓ orderId → Toss/아임웹 주문과 조인 가능
  
상담 DB (3,947건)
  ↓ 고객 전화번호, 검사유형(analysisType), 상담사
  ✕ 유입 채널/캠페인 정보 없음
  
Toss 결제
  ↓ orderId, status(DONE/CANCELED), amount
  ✕ 상품명/카테고리 없음 → 상품군별 환불률 불가
```

#### 산출 가능/불가 매트릭스

| 지표 | 전체 수준 | 캠페인별 | 상품군별 | 판정 |
|------|---------|---------|---------|------|
| **재구매율 (90/180일)** | ✅ 가동 중 (26.3%/28%) | ❌ 불가 | △ 검사유형별 가능 | 아래 상세 |
| **상담 연결률** | △ 전체는 추정 가능 | ❌ 불가 | △ 검사유형별 가능 | 아래 상세 |
| **환불/취소율** | ✅ 가능 (5.4%) | ❌ 불가 | ❌ 불가 | Toss에 상품정보 없음 |

#### 1. 캠페인별 재구매율 — 현재 불가, 해결책 있음

**왜 불가**: Meta는 "음식물 과민증 캠페인에서 298건 전환"이라고 알려주지만, 그 298명이 누구인지(전화번호/이메일)는 알려주지 않음. 누구인지 모르니 재구매 여부를 추적할 수 없음.

**해결 방법 (2가지)**:

| 방법 | 설명 | 난이도 | 정확도 |
|------|------|--------|--------|
| **A. UTM 기반 역추적** | attribution ledger의 `utm_campaign`으로 "이 주문은 어떤 캠페인에서 왔는가" 역추적 → orderId → 전화번호 → 재구매 조회 | 중 | 중 (현재 meta UTM 26건뿐, CAPI 적용 후 개선) |
| **B. 검사유형 proxy** | "음식물 과민증 캠페인 = 음식물 과민증 검사 구매자"로 가정 → 해당 검사유형의 재구매율을 캠페인 재구매율 proxy로 사용 | 낮 | 중하 (캠페인과 검사유형이 1:1 대응한다는 가정) |

**즉시 만들 수 있는 것**: 방법 B. callprice API의 `analysisType`별 전환율/매출/재구매율 이미 구조가 있음. 검사유형별 90일 전환율이 캠페인별 재구매율의 proxy.

#### 2. 캠페인별 상담 연결률 — 현재 불가, 간접 추정 가능

**왜 불가**: 상담 DB에 "이 고객이 어떤 광고를 보고 왔는가" 필드가 없음.

**해결 방법**:

| 방법 | 설명 | 난이도 |
|------|------|--------|
| **A. 전화번호 조인** | attribution ledger의 orderId → 아임웹 주문 → 전화번호 → 상담 DB 전화번호 매칭. "이 전화번호가 meta_biocom_* UTM으로 주문한 적이 있고, 상담도 받았다" | 높 |
| **B. 기간 proxy** | 특정 캠페인 집행 기간 동안의 상담 건수 변화를 관찰. 캠페인 시작 전/후 상담 건수 차이 = 광고 기여 상담 추정 | 중 |

**즉시 만들 수 있는 것**: 전체 상담 연결률 (상담 고객 수 / 전체 구매 고객 수). 현재 데이터 기준 상담 3,947건 / 구매 고객(추정) ~10,000명 = 약 **40%**.

#### 3. 환불/취소율 — 전체 가능, 캠페인/상품별 불가

**현재 가능한 것**:

| 사이트 | DONE | CANCELED | PARTIAL | 취소율 |
|--------|------|---------|---------|--------|
| 바이오컴 | 29,367건 | 1,452건 | 316건 | **5.4%** |
| 더클린커피 | 664건 | 20건 | 3건 | **3.3%** |

**캠페인별 불가 이유**: Toss 거래 데이터에 캠페인/유입 채널 정보가 없음. attribution ledger orderId와 Toss orderId를 조인하면 UTM이 있는 26건에 대해서만 "이 취소 건이 meta 캠페인 유입이었는가" 확인 가능하지만, 통계적 의미 없음.

**상품군별 불가 이유**: Toss에 상품명/카테고리 필드가 없음. 아임웹 주문에도 개별 상품 line-item이 없음 (total_price만 있음).

#### 4. 지금 만들 수 있는 것 (Claude Code 판단)

| # | 지표 | 기준 | 데이터 소스 | 난이도 |
|---|------|------|-----------|--------|
| 1 | **검사유형별 90일 전환율** | analysisType | callprice API | ✅ 기존 API 활용 |
| 2 | **전체 환불/취소율** | 사이트별 | Toss 로컬 캐시 | ✅ 이미 계산 완료 |
| 3 | **UTM별 주문 분포** | utm_source | attribution ledger | ✅ JSONL 파싱 |
| 4 | **채널별(meta/tiktok/naver) 주문 건수** | utm_source 그룹핑 | attribution ledger | ✅ 바로 가능 |
| 5 | **채널별 재구매율 proxy** | UTM 주문 → 전화번호 → 재구매 조회 | attribution + imweb_orders | 중 (표본 작음) |
| 6 | **전체 상담 연결률** | 상담 고객 / 구매 고객 | callprice + imweb | ✅ 바로 가능 |

#### 5. 만들 수 없는 것 (근본적 한계)

| 지표 | 불가 이유 | 해결 전제 |
|------|---------|---------|
| **캠페인별 재구매율** (정밀) | Meta가 전환 고객 개인정보를 안 줌 | CAPI 활성화 + UTM 100% 커버리지 |
| **캠페인별 상담 연결률** | 상담 DB에 유입 채널 없음 | 상담 예약 시 UTM 전달 or CRM에서 전화번호 역조인 |
| **상품군별 환불률** | Toss에 상품 정보 없음 | 아임웹 주문 line-item API 연동 |
| **캠페인별 환불률** | 위 두 가지 결합 | UTM + 상품 + Toss 상태 3-way 조인 |
