## 10초 요약

결론은 **알파리뷰 조건부 도입 추천**이다.

이유는 단순하다. 지금 필요한 것은 리뷰 UI 하나가 아니라 `리뷰 선별 노출`, `공동구매/비밀 링크 리뷰 연결`, `구매 후 리뷰 수집 자동화`, `관리 화면`, `적립금/알림톡 운영`까지 묶인 운영 기능이다. 이 범위를 직접 만들면 Meta 데이터 세트 제한 대응보다 더 큰 별도 프로젝트가 된다.

다만 알파리뷰가 Meta 데이터 세트 제한을 직접 해결하지는 않는다. `meta/!metaplan.md`의 핵심인 새 데이터 세트, 클린 랜딩, 민감 parameter 차단, 내부 confirmed ROAS 기준은 그대로 유지해야 한다. 알파리뷰는 클린 랜딩의 신뢰 보강 장치로만 사용한다.

## 현재 문제와 연결

`meta/!metaplan.md`의 현재 결론은 아래다.

- 기존 데이터 세트는 끄지 않는다.
- 신규 클린 데이터 세트와 기존 세트를 제한적으로 병행한다.
- 새 랜딩은 건강 상태/증상/검사 상세값을 이벤트 parameter로 보내지 않는다.
- 내부 예산 판단은 Meta 플랫폼 구매값이 아니라 Imweb UTM, GA4 UTM, VM Cloud/운영DB confirmed order로 본다.

알파리뷰는 이 중 `신규 랜딩` 쪽과 연결된다.

연결되는 부분:

- 새 랜딩에서 제품 설명만으로 부족한 신뢰도를 실제 고객 경험으로 보강한다.
- 공동구매/인플루언서 비밀 링크에서 쌓인 리뷰를 본상품 또는 리뷰 중심 랜딩에 연결할 수 있다.
- 리뷰 수집을 알림톡 기반으로 자동화해 리뷰 자산을 계속 늘린다.

연결되지 않는 부분:

- Meta 데이터 소스 카테고리 제한 자체를 풀지는 않는다.
- Pixel/CAPI 정책 리스크를 없애지 않는다.
- 광고 최적화 신호를 자동으로 복구하지 않는다.
- 잘못된 리뷰 문구가 노출되면 오히려 건강/검사 민감 표현 리스크가 커질 수 있다.

## Notion 문서 핵심

Notion `알파리뷰 도입 검토 보고서`의 제안은 아래로 요약된다.

- 추진 내용: 홈 메인 리뷰 선별 노출 + 비밀 링크 리뷰 연동
- 추진 배경: 자사몰 리뷰 부족, 공동구매/비밀 링크 채널 리뷰가 자사몰에 쌓이지 않음
- 제안 조건: 2026년 5월 내 계약 시 8주 무료 체험, 12개월 선납 월 60,000원, 연 720,000원
- 비교 대상: 크리마 월 110,000원, 알파리뷰 월 120,000원 정가
- 주요 기능: 빠른리뷰, 키워드 리뷰, 설문 리뷰, 리뷰 위젯, 상품 간 리뷰 연결

이 내용은 방향이 맞다. 특히 현재 Meta 유입 랜딩은 리뷰/고객 경험이 중요한데, 리뷰가 흩어져 있으면 새 랜딩을 만들어도 전환율을 방어하기 어렵다.

다만 Notion 문서에서 보강해야 할 점이 있다.

- 알파리뷰가 Meta 데이터 제한 해결책처럼 보이면 안 된다.
- 공식 가격 기준으로는 최근 1개월 주문 건수에 따라 플랜이 자동 책정된다.
- 알림톡/트래픽 초과 비용은 별도 후불이다.
- 아임웹에서는 일부 기능 제한이 있다는 공식 문구가 있으므로, 필요한 기능 2개가 실제로 아임웹에서 되는지 계약 전 확인해야 한다.

## 공식 가이드 확인 메모

알파리뷰 공식 가이드 기준 확인 사항:

- 아임웹 플랜은 최근 1개월 주문 건수 기준으로 자동 책정된다.
- 비즈니스 플랜은 5,000건 이하 기준 월 120,000원이다.
- 알림톡/트래픽 초과 비용은 서비스 이용료와 별도로 청구된다.
- 알림톡은 일반적으로 주문 1건당 1건, 월 주문 건수 x 12원 수준으로 안내된다.
- 트래픽 비용은 1GB당 600원, 월 방문자 10만 명당 대략 5,000~10,000원으로 안내된다.
- 아임웹에서는 디자인 모드에서 위젯을 직접 추가해야 한다.
- 상품 간 리뷰 연결 기능은 그룹 연결/단방향 연결을 지원한다.

아임웹 크리마 공식 가이드 기준 확인 사항:

- 크리마도 아임웹 연동은 가능하다.
- 단일 크리마 계정은 단일 아임웹 사이트만 연동된다.
- 크리마 리뷰 사용 중에는 구매평 조회/관리/설정이 크리마 관리자 중심으로 이동한다.

## 직접 구축 시 범위

직접 구축은 범위를 작게 자르면 가능하다.

### 직접 구축 MVP

목표:

- 홈 또는 Meta 클린 랜딩에 선별 리뷰 5~20개 노출
- 리뷰 카드 UI
- 운영자가 수정할 수 있는 JSON/Markdown 또는 간단 CMS
- GA4/내부 이벤트로 리뷰 영역 노출, 클릭, 스크롤만 측정

예상:

- 1~3일
- 운영 리뷰 수동 입력
- 리뷰 수집/적립금/알림톡/비밀 링크 연동 없음

장점:

- Meta 클린 랜딩 정책에 맞춰 문구와 이벤트를 완전히 통제할 수 있다.
- 외부 스크립트와 개인정보 전송 리스크가 작다.
- 빠르게 실험 가능하다.

한계:

- 리뷰가 계속 쌓이지 않는다.
- 공동구매/비밀 링크 리뷰를 자동으로 연결하지 못한다.
- 운영자가 계속 수동 관리해야 한다.
- 리뷰 진위/작성자/주문 검증/혜택 지급까지 직접 만들면 MVP가 아니다.

### 직접 구축 풀버전

필요 기능:

- 주문 기반 리뷰 요청 발송
- 알림톡/SMS 연동
- 리뷰 작성 페이지
- 포토/동영상 업로드
- 적립금 지급
- 검수/차단/신고/댓글
- 상품 간 리뷰 연결
- 비밀 링크 상품과 본상품 리뷰 매핑
- 리뷰 위젯 관리
- 아임웹 상품/주문/회원 데이터 동기화
- 개인정보 처리/보관/삭제 정책
- 관리자 UI와 권한
- SEO/noindex/canonical 정책

판단:

- 최소 3~6주 이상 별도 프로젝트다.
- 운영/CS/개인정보 책임까지 생긴다.
- 지금 Meta 데이터 세트 제한 대응보다 우선하면 안 된다.

## 알파리뷰 도입 시 범위

알파리뷰가 대신 처리하는 부분:

- 리뷰 위젯
- 선별 리뷰 노출
- 빠른리뷰 알림톡
- 키워드/설문 리뷰
- 상품 간 리뷰 연결
- 리뷰 관리 UI
- 포인트 지급 설정

우리 쪽에서 해야 하는 부분:

- 클린 랜딩에 어떤 리뷰를 노출할지 기준 설정
- 민감 표현 리뷰 제외 기준
- 아임웹 디자인 모드 위젯 삽입
- 위젯 성능/스크립트/추적 요청 검수
- Pixel/CAPI 이벤트 payload에 리뷰 내용, 상품명, 증상/검사명 원문이 섞이지 않는지 검수
- 내부 UTM/confirmed ROAS와 전환율 비교

## 비용과 손익 기준

Notion 제안 기준:

- 할인 조건: 월 60,000원
- 연간 선납: 720,000원
- VAT 별도 가능성 있음

공식 정가 기준:

- 비즈니스 플랜: 월 120,000원
- 알림톡/트래픽 초과 비용 별도

2026-05-18 `meta/!metaplan.md`의 Imweb 캡처 기준:

- Meta 유료 FB/IG 주말 방문자: 14,305명
- 구매자: 44명
- 매출: 15,663,000원
- 구매전환율: 0.31%
- 객단가: 355,977원

단순 손익 해석:

- 월 60,000원 조건이면 월 추가 주문 1건만 만들어도 대부분의 마진 구조에서 비용 회수가 가능하다.
- 월 120,000원 정가여도 객단가 355,977원 기준 추가 주문 1~2건이면 비용 회수 가능성이 높다.
- 따라서 비용 자체는 큰 문제가 아니다.

주의:

- 실제 판단은 매출이 아니라 공헌이익 기준이어야 한다.
- 월 주문 수가 5,000건을 넘으면 플랜이 자동 상향될 수 있다.
- 알림톡/트래픽 비용과 리뷰 적립금 비용은 별도다.

## 추천 결론

추천은 **알파리뷰 조건부 도입 + 직접 구축 MVP 보완**이다.

실행 순서:

1. 알파리뷰를 8주 무료 체험 또는 계약 전 검증으로 시작한다.
2. 아임웹에서 `홈 메인 선별 리뷰`, `상품 간 리뷰 연결`, `비밀 링크 상품 리뷰 공유`, `위젯 설치`, `빠른리뷰 알림톡`이 실제로 가능한지 체크한다.
3. Meta 클린 랜딩에는 알파리뷰 위젯을 바로 전체 삽입하지 말고, 먼저 선별 리뷰 위젯 또는 정적 리뷰 섹션으로 제한 테스트한다.
4. 리뷰 문구에서 건강 상태, 증상, 질환, 검사 결과 원문, 효능 보장 표현을 제외한다.
5. 성능과 추적 요청을 확인한 뒤 운영 랜딩에 확장한다.
6. 직접 구축은 알파리뷰 대체가 아니라 `정책 안전한 리뷰 섹션`, `전환 측정`, `랜딩별 리뷰 노출 정책`만 맡는다.

연간 선납 판단:

- 8주 무료와 월 60,000원 조건이 계약서/청구서에 명확히 적히면 12개월 선납은 비용상 타당하다.
- 다만 `아임웹에서 필요한 기능 2개가 실제 작동`, `위젯 성능 문제 없음`, `민감 표현 필터링 가능`을 확인하기 전에는 카드 결제/선납을 보류한다.

## 계약 전 확인 질문

알파리뷰 담당자에게 아래를 확인해야 한다.

1. 아임웹에서 홈 메인에 원하는 리뷰만 선별 노출할 수 있는가?
2. 아임웹에서 비밀 링크 상품 리뷰를 본상품 또는 다른 상품에 연결할 수 있는가?
3. 상품 간 리뷰 연결이 URL 단위인지, 상품 ID 단위인지, 둘 다 가능한지?
4. 공동구매/인플루언서 전용 상품을 삭제하거나 숨겨도 리뷰 연결이 유지되는가?
5. 리뷰 문구의 특정 키워드 제외, 고정, 숨김, 승인 워크플로가 가능한가?
6. 리뷰 위젯 스크립트가 어떤 외부 도메인으로 요청을 보내는가?
7. 알림톡 발송 주체, 비용, 발송 거절/수신동의 처리는 어떻게 되는가?
8. 리뷰 데이터 CSV export와 계약 종료 후 이관이 가능한가?
9. 8주 무료 체험과 월 60,000원 12개월 조건이 서면으로 보장되는가?
10. 아임웹에서 제한되는 기능이 무엇인가?

## Meta 클린 랜딩 적용 기준

리뷰 위젯을 Meta 클린 랜딩에 넣을 때는 아래 기준을 지킨다.

허용:

- 배송/상담/구매 경험
- 사용 편의성
- 설명이 이해 쉬웠다는 후기
- 결과 확인 과정이 편했다는 수준의 일반 경험
- 고객 만족도, 재구매 의향

주의 또는 제외:

- 특정 질환명
- 증상 개선 주장
- 검사 결과 상세값
- 민감한 건강 상태를 드러내는 문장
- 효능 보장 표현
- 개인 식별 정보

이유:

새 데이터 세트와 클린 랜딩의 목적은 우회가 아니라 불필요한 민감 데이터 전송을 줄이는 것이다. 리뷰가 민감 표현으로 가득하면 새 랜딩도 다시 제한될 수 있다.

## 최종 판정

판정: **진행 추천**

추천 강도: 84%

조건:

- 알파리뷰를 도입하되 Meta 제한 해결책으로 포장하지 않는다.
- 8주 무료 체험 또는 설치 검증에서 핵심 기능 2개를 먼저 확인한다.
- Meta 클린 랜딩에는 리뷰 노출을 선별 적용한다.
- 직접 구축은 리뷰 솔루션 전체가 아니라 랜딩/측정/정책 통제 보완만 한다.

## 하지 않은 것

- 알파리뷰 계약/결제를 하지 않았다.
- 아임웹 앱 설치를 하지 않았다.
- 리뷰 위젯 스크립트를 운영 사이트에 넣지 않았다.
- Meta/GTM/Pixel/CAPI 설정을 바꾸지 않았다.
- 운영DB/VM Cloud/로컬 DB에 write하지 않았다.
