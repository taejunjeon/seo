## 한 줄 요약

클릭→랜딩뷰 이탈률은 **link_clicks 기준 37.1%**(clicks 기준 43.6%에서 6.5%p 개선). 이탈의 주 원인은 **모바일 LCP 36.6초**(Performance 27점)이고, Hotjar/Clarity로는 이 구간을 잡을 수 없다. **먼저 페이지 속도를 고치고, Clarity로 랜딩뷰→구매 전환율을 올려야** 한다.

## 1. 현재 퍼널 현황

### 클릭 유형 정정 — clicks vs link_clicks

| 지표 | clicks (all) | link_clicks | 차이 |
|------|-------------|-------------|------|
| 건수 | 181,076 | **162,486** | 18,590건 차이 (좋아요/댓글/공유 포함) |
| → 랜딩뷰 | 102,183 | 102,183 | 동일 |
| **이탈률** | **43.6%** | **37.1%** | 6.5%p 차이 |

**결론**: 기존 43.6% 이탈률은 과대평가됨. **link_clicks 기준 37.1%가 실제.** 하지만 37%도 업계 평균(20~30%)보다 높음.

### 캠페인별 link_clicks → 랜딩뷰 전환율

| 캠페인 | link_clicks | 랜딩뷰 | 전환율 | 판정 |
|--------|-----------|--------|--------|------|
| 호르몬 검사 전환 | 19,229 | 15,263 | **79%** | 양호 |
| 건강기능식품 | 5,933 | 4,475 | **75%** | 양호 |
| 호르몬 바이럴 | 1,948 | 1,376 | **71%** | 보통 |
| 영양중금속검사 | 13,766 | 9,559 | **69%** | 보통 |
| **음식물 과민증 (10/14~)** | **32,633** | **17,345** | **53%** | **문제** |

**핵심 발견**: 음식물 과민증 캠페인이 link_clicks→랜딩뷰 53%로 가장 낮음. 이 캠페인만 개선해도 전체 이탈률이 크게 줄어듦.

### 3사이트 모바일 PageSpeed 비교 (0406 실측)

| 사이트 | Performance | LCP | FCP | TTFB | link→land | 판정 |
|--------|------------|-----|-----|------|-----------|------|
| **바이오컴** | **32점** | **42.4초** | 21.2초 | 130ms | 62.9% | **치명적** — LCP가 가장 느리고 이탈률도 최악 |
| **더클린커피** | **36점** | **11.5초** | 5.9초 | 120ms | 미확인 | **나쁨** — 바이오컴보다 3.7배 빠르지만 권장(2.5초)의 4.6배 |
| **AIBIO** | 측정 실패 | — | — | — | **91.5%** | **양호 추정** — 이탈률이 가장 낮아 페이지가 가벼울 가능성 |

**LCP와 link→land 이탈률의 상관관계**:
- 바이오컴: LCP 42.4초 → 이탈 37.1% (최악)
- AIBIO: (측정 실패, 추정 빠름) → 이탈 8.5% (최고)
- **LCP가 빠를수록 이탈률이 낮아지는 명확한 패턴**

**바이오컴 병목 상세** (2026-02 측정 기준):
- 서버(TTFB 130ms)는 빠르지만, **프론트 리소스가 병목**
- 363개 요청, 26.3MB, 스크립트 154개
- 렌더 블로킹: 아임웹 site.css, jQuery, 네이버 WCS, 카카오 KAS, Vimeo 등 8개+

## 2. Microsoft Clarity란?

### 한 줄 설명

Microsoft가 만든 **무료 UX 분석 도구**. 세션 리플레이(무제한), 히트맵, 스크롤 맵, Dead click/Rage click 감지를 제공. Hotjar와 비슷하지만 **완전 무료 + 세션 제한 없음**.

### Hotjar와 비교

| 항목 | Microsoft Clarity | Hotjar Free | Hotjar Business |
|------|------------------|-------------|----------------|
| 가격 | **무료** | 무료 | 월 $80+ |
| 세션 리플레이 | **무제한** | 35/일 | 유료 |
| 히트맵 | ✅ | ✅ | ✅ |
| Dead click 감지 | ✅ (자동) | ❌ | 유료 |
| Rage click 감지 | ✅ (자동) | ❌ | 유료 |
| AI Copilot | ✅ | ❌ | 유료 |
| 설문/피드백 위젯 | ❌ | ✅ | ✅ |
| GA4 연동 | ✅ | ✅ | ✅ |
| 스크립트 무게 | **가벼움** (~17KB) | 중간 (~35KB) | 중간 |

### Clarity의 핵심 기능

| 기능 | 설명 | 랜딩뷰 개선에 도움? |
|------|------|------------------|
| **세션 리플레이** | 실제 사용자가 페이지에서 어떻게 행동했는지 영상으로 재생 | ✅ 어디서 이탈하는지 정확히 파악 |
| **히트맵** | 사용자가 클릭하는 영역을 색상으로 시각화 | ✅ CTA 버튼 효과 검증 |
| **스크롤 맵** | 페이지의 어느 지점까지 스크롤하는지 표시 | ✅ CTA가 보이는 위치까지 스크롤하는 비율 확인 |
| **Dead clicks** | 클릭했지만 반응 없는 UI 요소 자동 감지 | ✅ 깨진 링크, 비활성 버튼 발견 |
| **Rage clicks** | 같은 곳을 반복 클릭하는 패턴 감지 (사용자 좌절) | ✅ UX 문제 자동 발견 |
| **AI Copilot** | "이 페이지에서 가장 큰 UX 문제가 뭔가?" 질문하면 AI가 답변 | ✅ 분석 시간 절약 |
| **필터링** | 디바이스, OS, 페이지, UTM, 이벤트별 필터 | ✅ 캠페인별 행동 차이 분석 |

## 3. Clarity 설치 방법 (아임웹 + GTM)

### 사전 준비 (5분)

1. https://clarity.microsoft.com 접속
2. Microsoft 계정으로 로그인 (없으면 무료 생성)
3. "New Project" 클릭
4. 사이트 URL 입력: `https://biocom.kr`
5. **Project ID** 복사 (예: `abc123xyz`)

### GTM으로 설치 (권장 — 이미 GTM이 있으므로)

1. https://tagmanager.google.com 접속
2. 바이오컴 GTM 컨테이너 선택
3. **"태그" > "새로 만들기"** 클릭
4. 태그 유형: **"맞춤 HTML"**
5. HTML에 아래 코드 붙여넣기:

```html
<script type="text/javascript">
    (function(c,l,a,r,i,t,y){
        c[a]=c[a]||function(){(c[a].q=c[a].q||[]).push(arguments)};
        t=l.createElement(r);t.async=1;t.src="https://www.clarity.ms/tag/"+i;
        y=l.getElementsByTagName(r)[0];y.parentNode.insertBefore(t,y);
    })(window, document, "clarity", "script", "여기에_Project_ID_입력");
</script>
```

6. 트리거: **"Window Loaded"** (페이지 로드 완료 후 실행 → Core Web Vitals 영향 최소화)
   - 트리거가 없으면 "트리거 구성" > "페이지 뷰" > "Window Loaded" 선택
7. **"저장"** > **"제출"** (게시)

### ✅ 3사이트 설치 완료 (0406)

| 사이트 | Clarity 프로젝트 | GTM 통합 | GA4 연동 | 봇 감지 |
|--------|---------------|---------|---------|--------|
| 바이오컴 (biocom.kr) | ✅ | ✅ | ✅ | ✅ |
| 더클린커피 (thecleancoffee.com) | ✅ | ✅ | 확인 필요 | ✅ |
| AIBIO (aibio.kr) | ✅ | ✅ | 확인 필요 | ✅ |

30분~1시간 후 각 사이트의 Clarity 대시보드에 데이터 유입 시작.

## 5. 클릭→랜딩뷰 이탈 37% 개선 방안

### Claude Code 의견

#### 이탈 원인 우선순위

```
1순위: 페이지 로딩 속도 (LCP 36.6초) ← 가장 큰 원인
2순위: 렌더 블로킹 리소스 363개 요청 / 26.3MB
3순위: 캠페인별 랜딩 URL 불일치 (음식물 과민증 53%)
4순위: 봇/무효 트래픽 (Meta 광고 특성상 5~10% 존재)
```

#### 아임웹에서 할 수 있는 속도 개선

아임웹은 서버를 직접 제어할 수 없지만, 프론트 단에서 개선 가능한 것:

| # | 액션 | 예상 효과 | 난이도 |
|---|------|---------|--------|
| 1 | **아임웹 관리자 > 불필요한 앱/위젯 제거** | 요청 수 20~30% 감소 | 낮음 |
| 2 | **이미지 최적화**: WebP 변환, 해상도 조정 | 전송량 30~50% 감소 | 낮음 |
| 3 | **Vimeo 플레이어 lazy loading**: 영상이 스크롤 후에만 로드 | LCP 개선 | 중간 |
| 4 | **네이버 WCS/카카오 KAS 비동기 로딩**: GTM에서 Window Loaded 이후 트리거로 이동 | 렌더 블로킹 2개 제거 | 중간 |
| 5 | **Beusable 중복 제거** (더클린커피에 2번 설치됨) | 불필요한 요청 제거 | 낮음 |
| 6 | **광고 랜딩 URL을 특정 상품 페이지로 직접 연결** (메인 대신): 음식물 과민증 → 해당 검사 상세 페이지 | 랜딩뷰 전환율 10~20%p 개선 | 중간 |

#### 캠페인별 차별 전략

```
음식물 과민증 (link→landing 53%):
  → 현재 랜딩 URL이 메인 페이지일 가능성.
  → 해당 검사 상품 상세 페이지로 직접 연결하면 70%+ 기대.
  → 광고 소재의 메시지("음식물 알러지 검사")와 랜딩 첫 화면이 일치하는지 점검.

호르몬 검사 (link→landing 79%):
  → 이미 양호. 이 캠페인의 랜딩 구조를 다른 캠페인에도 적용.
```

#### 경량 랜딩 페이지 검토

아임웹 외부에 별도 경량 랜딩 페이지를 만드는 것도 방법:
- Next.js/Vercel로 1페이지 랜딩 제작 → LCP 1~2초 달성 가능
- 광고에서 이 경량 페이지로 유입 → 상세 정보 원하면 biocom.kr로 이동
- A/B 테스트: 기존 biocom.kr vs 경량 랜딩 → 전환율 비교

### Codex 의견

#### 1순위: link_clicks 기준으로 리포트 분모 교체

현재 대시보드에서 `clicks(all) = 181,076`을 분모로 쓰고 있는데, 이 중 18,590건은 좋아요/댓글/공유. **link_clicks = 162,486을 분모로 써야** 실제 이탈률이 정확함.

수정 전: 이탈률 43.6%
수정 후: 이탈률 **37.1%** (6.5%p 감소, 실제로는 이만큼 덜 나쁨)

#### 2순위: 모바일 LCP 36.6초 해결

내부 측정(lcpfcp1.0.md) 기준 TTFB 106ms → 서버는 빠름. 병목은:
- 렌더 블로킹 CSS 3개 (아임웹 site.css, vendor_blue_10.css, custom.css) → 합산 10초+ 차단
- 렌더 블로킹 JS (jQuery, 네이버 WCS, Vimeo) → 합산 4초+ 차단
- 이미지/폰트 26.3MB → 초기 로드에 모두 요청

아임웹에서 서버 설정은 못 건드리지만, GTM에서 서드파티 스크립트의 로딩 순서를 제어하면 **체감 LCP를 10초 이하로 줄이는 것은 가능**.

#### 3순위: 봇/무효 트래픽 추정

Meta 광고에서 봇 비중은 일반적으로 **5~15%**. 바이오컴 월 16만 link_clicks 중 5% = 약 8,000건이 봇일 수 있음. 이 경우:
- 실제 human link_clicks: ~154,000
- 실제 이탈률: (154,000 - 102,183) / 154,000 = **33.6%** (업계 평균에 근접)

봇 필터링은 GA4에서 "봇 필터링" 설정 확인 + Clarity의 "비정상 세션" 필터로 확인 가능.

## 6. 랜딩뷰→구매 1.25% 전환율 개선 방안

### Claude Code 의견

| # | 액션 | 예상 효과 | Clarity 활용 |
|---|------|---------|------------|
| 1 | **CTA 위치 최적화**: ATF(스크롤 없이 보이는 영역)에 "검사 신청하기" 버튼 배치 | 전환율 0.2~0.5%p 향상 | 스크롤 맵으로 현재 CTA 위치까지 도달하는 비율 확인 |
| 2 | **사회적 증거 강화**: "이미 OO,OOO명이 검사했습니다" + 후기 썸네일 | 전환율 0.3~0.5%p | 히트맵으로 리뷰 섹션 클릭률 확인 |
| 3 | **모바일 결제 UX**: 아임웹 기본 결제 플로우에서 불필요한 단계 제거 | 전환율 0.1~0.3%p | 세션 리플레이에서 결제 이탈 시점 확인 |
| 4 | **가격 앵커링**: "정가 ₩500,000 → 특가 ₩350,000" 형태로 할인 강조 | 전환율 0.2~0.4%p | A/B 테스트 후 Clarity에서 행동 차이 비교 |
| 5 | **긴급성 부여**: "이번 주 한정 무료 상담" 같은 제한 시간 오퍼 | 전환율 0.1~0.3%p | 구매 vs 이탈 세션의 체류 시간 비교 |

### Codex 의견

랜딩뷰→구매 1.25%는 "보통 범위(1~3%)"이지만, 검사키트 AOV ₩462,000 감안 시 **고관여 상품으로서는 양호**. 개선 여지는 있지만 급격한 개선보다는 **점진적 A/B 테스트**가 안전.

Clarity 설치 후 **첫 번째로 볼 것**: "구매 완료한 세션 vs 장바구니에서 이탈한 세션"의 행동 차이. 이 비교에서 가장 많이 나오는 이탈 지점이 1순위 개선 대상.
