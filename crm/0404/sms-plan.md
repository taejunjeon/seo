# CRM 관리 허브 — SMS 문자 발송 기능 구현 계획

> **기준일**: 2026-04-04
> **위치**: Phase 3 확장 (P3-S6)
> **담당**: Codex (백엔드) + Claude Code (프론트)

## 한 줄 요약

백엔드 SMS 발송은 **이미 완성**돼 있다. 프론트에 채널 선택 UI만 추가하면 CRM에서 알림톡/SMS 모두 발송 가능.

## 알림톡 vs SMS 차이점

| | 알림톡 (카카오) | SMS (문자) |
|---|-------------|-----------|
| **API URL** | `kakaoapi.aligo.in` | `apis.aligo.in` |
| **메시지 형태** | 승인된 템플릿 필수 | 자유 입력 |
| **글자 수** | ~1,000자 | 90바이트 (한글 45자, 초과 시 LMS) |
| **버튼** | 최대 5개 | 불가 |
| **변수 치환** | `#{이름}` 등 | 불필요 |
| **수신 동의** | 카카오 채널 구독 | SMS 마케팅 동의 (`marketing_agree_sms`) |
| **비용** | 건당 ~8원 | 건당 ~16원 (SMS) / ~45원 (LMS) |
| **도달률** | 카카오톡 설치자만 | 모든 휴대폰 |

### 정보성 vs 홍보성

| | 정보성 | 홍보성 |
|---|-------|-------|
| **알림톡** | consent 불필요 | consent 필요 + 080 수신거부 |
| **SMS** | consent 불필요 (거래/안내) | `marketing_agree_sms = Y` 필수 + 080 |

## 구현 계획

### Codex 담당 (백엔드)

#### P3-S6-B1: SMS 전용 contact policy 확장

현재 `evaluateContactPolicy()`는 `channel: "aligo"` (알림톡)만 구분함. SMS 채널 추가 필요.

```typescript
// contactPolicy.ts 수정
channel: "aligo" | "channeltalk" | "sms"

// SMS 전용 규칙:
// - consent: marketing_agree_sms = "Y" (홍보성일 때)
// - quiet hours: 동일 (21:00-08:00 KST)
// - cooldown: 24h (동일)
// - frequency cap: SMS max 2/week (동일)
// - 080 수신거부번호 안내 필수 (홍보성)
```

#### P3-S6-B2: SMS 글자 수 판정 API

```
POST /api/aligo/sms-preview
  Body: { "message": "안녕하세요..." }
  Response: {
    "byteLength": 42,
    "type": "SMS",        // SMS(90바이트 이하) 또는 LMS(90바이트 초과)
    "estimatedCost": 16,  // 원
    "charCount": 21
  }
```

#### P3-S6-B3: SMS 발신번호 확인 API (선택)

```
GET /api/aligo/sms-sender
  Response: { "sender": "01012345678", "verified": true }
```

### Claude Code 담당 (프론트)

#### P3-S6-F1: MessagingTab 채널 선택 UI

현재 MessagingTab 상단에 채널 토글 추가:

```
┌──────────────────────────────────────────────┐
│  ┌──────────────┐  ┌──────────────┐          │
│  │ 카카오 알림톡  │  │  SMS 문자    │          │
│  └──────────────┘  └──────────────┘          │
│                                              │
│  [채널에 따라 다른 UI 표시]                    │
└──────────────────────────────────────────────┘
```

**알림톡 선택 시**: 기존과 동일 (템플릿 → 변수 → 미리보기 → 발송)

**SMS 선택 시**: 아래 새로운 흐름

```
Step 1: 수신자 선택 (기존과 동일)
  ↓
Step 2: 메시지 작성
  ┌─────────────────────────────────────────┐
  │  메시지 내용                             │
  │  ┌─────────────────────────────────────┐│
  │  │ 안녕하세요 #{이름}님,                ││
  │  │ 더클린커피에서 새 원두가 입고됐습니다. ││
  │  │                                     ││
  │  └─────────────────────────────────────┘│
  │  42/90 바이트 (SMS)  예상 비용: ₩16      │
  │  ※ 90바이트 초과 시 LMS(₩45)로 전환      │
  └─────────────────────────────────────────┘
  ↓
Step 3: 정책 확인 + 발송
  ┌─────────────────────────────────────────┐
  │  ✅ SMS 수신 동의 확인                    │
  │  ✅ 발송 가능 시간 (08:00~21:00)         │
  │  ✅ 24시간 쿨다운 통과                    │
  │                                         │
  │  [테스트 발송]  [실제 발송]               │
  └─────────────────────────────────────────┘
```

#### P3-S6-F2: SMS 글자 수 실시간 카운터

- 입력 시 바이트 수 실시간 표시 (한글 2바이트, 영문 1바이트)
- 90바이트 이하: "SMS (₩16)" 표시
- 90바이트 초과: "LMS (₩45)" 경고 표시
- 프론트에서 계산 가능 (백엔드 호출 불필요)

#### P3-S6-F3: 재구매 탭 → SMS 발송 연결

현재 재구매 탭의 "알림톡 발송" 버튼 옆에 "SMS 발송" 버튼 추가:

```
[SMS 동의 고객 22명에게 알림톡 발송 →]  [SMS 발송 →]
```

## 실행 순서

```
1단계: 백엔드 정비 (Codex)
├── P3-S6-B1: contact policy에 "sms" 채널 추가

2단계: 프론트 구현 (Claude Code)
├── P3-S6-F1: MessagingTab 채널 토글 + SMS 입력 UI
├── P3-S6-F2: 글자 수 실시간 카운터
└── P3-S6-F3: 재구매 탭 SMS 발송 버튼

3단계: 검증
├── 화이트리스트 대상 테스트 발송
└── consent 체크 동작 확인
```

## 의존 관계

```
[이미 완료]                          [새로 구현]
POST /api/aligo/sms ──────────→ MessagingTab 채널 토글 (F1)
sendAligoSms() ────────────────→ SMS 메시지 입력 UI (F1)
발송 로그 JSONL ────────────────→ (변경 불필요, channel:"sms" 이미 기록)
contact policy ─── B1 확장 ───→ SMS 전용 정책 검증 (F1)
```

## 리스크

| 리스크 | 대응 |
|--------|------|
| SMS 비용이 알림톡보다 2배 (₩16 vs ₩8) | UI에 예상 비용 표시, LMS 전환 경고 |
| 홍보성 SMS는 080 수신거부번호 필수 | 메시지 끝에 자동 추가 로직 |
| 알림톡 실패 시 SMS fallback 요구 가능 | 현재는 별도 채널로 운영, fallback은 Phase 7 이후 |
