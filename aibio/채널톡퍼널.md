## 10초 요약
- AIBIO 홈페이지의 채널톡 버튼 클릭과 메신저 열림은 Meta 전환 퍼널의 일부로 잡을 수 있다.
- 이미 `GTM-T8FLZNT` 게시본에는 `aibio_channeltalk_open` 이벤트가 있고, GA4와 Meta Pixel `trackCustom`으로 보내는 구조가 들어 있다.
- 2026-04-24 TJ님 캡처 기준 Meta Pixel Helper와 Events Manager에서 `aibio_channeltalk_open` 수신이 확인됐다. 이제 "설치 가능성"이 아니라 "브라우저 이벤트 수신 확인" 상태다.
- TJ님 확인 기준 맞춤 이벤트 확인도 완료됐다. Custom Conversion과 30일 Website Custom Audience도 API로 생성됐고, 2026-04-25 광고관리자 컬럼도 추가됐다. 이제 남은 일은 실제 Lead/상담 전환율 비교다.
- 다만 이 이벤트는 `Lead`가 아니라 상담 의도에 가까운 마이크로 전환이다. 광고 최적화의 1차 목표는 계속 `aibio_form_submit -> Lead`로 둔다.
- Meta의 전환 API 권고는 채널톡 open의 즉시 blocker가 아니다. 다만 `Lead`, `Schedule`, `InitiateCheckout`를 광고 최적화에 쓸 때는 CAPI mirror를 붙이는 쪽이 맞다.
- 광고 화면의 "머신러닝 제한됨"은 지금 당장 설정을 갈아엎을 신호가 아니다. 최근 30일 Lead가 13건이라 학습 종료에 필요한 신호량이 부족한 것이 핵심이다.
- 2026-04-25 13:20 KST Meta API 기준 활성 캠페인은 2개다. `26.01.16 리커버리랩 예약 캠페인`은 트래픽 캠페인, `리드 캠페인 (소재복사 0406_2352)`은 리드 캠페인이 맞다.
- 최근 30일 API 기준 트래픽 캠페인에는 `Lead`와 `AIBIO - ChannelTalk Open` 전환이 보고되지 않았다. 리드 캠페인에는 `Lead` 15건이 보고된다.

## 문서 목적
이 문서는 AIBIO 홈페이지 채널톡 버튼을 Meta 광고 퍼널에서 어떤 의미로 기록하고, 어디까지 전환 최적화에 써야 하는지 정리한다.

## 이 작업이 하는 일
이 작업은 `광고 클릭 -> 홈페이지 방문 -> 채널톡 열기 -> 실제 상담 시작 -> 방문 예약` 흐름을 분리해서 기록하는 일이다.

채널톡 버튼을 누른 사람은 폼 제출자보다 약한 신호지만, 단순 방문자보다 강한 신호다. 따라서 이 이벤트는 리타겟팅, 광고 소재 비교, 랜딩 개선 판단에 쓸 수 있다.

## 왜 필요한가
AIBIO는 구매 쇼핑몰이 아니라 오프라인 센터 리드 사업이다. 폼 제출만 보면 리드 수는 보이지만, 사용자가 상담을 시도하려는 중간 행동은 놓친다.

채널톡 버튼 클릭을 잡으면 아래 질문에 답할 수 있다.

- 광고 유입자가 페이지를 보고 바로 이탈했는가, 아니면 상담 창까지 열었는가
- 폼 제출은 안 했지만 상담 의도가 있는 사용자가 얼마나 되는가
- 카카오 상담, 채널톡, 폼 제출 중 어떤 CTA가 더 강하게 작동하는가
- 채널톡을 연 사람에게 다시 광고를 보여줄 가치가 있는가

## 결론

### 현재 조건에서 가능한 것
Meta에서 채널톡 버튼 클릭을 전환 퍼널 일부로 잡는 것은 가능하다.

이미 AIBIO GTM 게시본에는 아래 구조가 있다.

| 항목 | 값 |
|---|---|
| GTM 정본 | `GTM-T8FLZNT` |
| Meta Pixel ID | `1068377347547682` |
| 현재 채널톡 이벤트 | `aibio_channeltalk_open` |
| GA4 태그 | `GA4 aibio_channeltalk_open` |
| Meta 태그 | `Meta Custom - aibio_channeltalk_open` |
| Meta 전송 방식 | `fbq('trackCustom')` |
| Meta 화면 검증 | Pixel Helper와 Events Manager에서 수신 확인 |
| 이벤트 성격 | 마이크로 전환, 리타겟팅 신호, 퍼널 진단 신호 |

### 아직 하면 안 되는 것
채널톡 버튼 클릭을 바로 Meta `Lead`로 보내면 안 된다.

버튼 클릭은 상담창을 연 행동이다. 상담을 실제로 시작했는지, 연락처를 남겼는지, 방문 예약으로 이어졌는지와 다르다. 이걸 `Lead`로 보내면 Meta가 실제 리드보다 넓은 행동을 리드로 학습할 수 있다.

### 추천 기준
현재는 아래처럼 나눈다.

| 사용자 행동 | Meta 이벤트 | 용도 | 최적화 목표로 사용 |
|---|---|---|---|
| 채널톡 버튼 클릭 또는 메신저 열림 | `aibio_channeltalk_open` custom event | 리타겟팅, 퍼널 진단 | 아직 보류 |
| 채널톡에서 새 상담방 생성 | `aibio_channeltalk_chat_created` custom event | 상담 의도 측정 | 데이터 축적 후 검토 |
| 연락처를 남기거나 상담 신청 완료 | `Lead` standard event | 광고 최적화 | 가능 |
| 방문 예약 확정 | `Schedule` standard event | 고품질 리드 최적화 | 방문 예약 원장 생긴 뒤 가능 |

## 현재 상태

### 확인된 것
- 라이브 HTML에서 Meta Pixel `1068377347547682`가 확인됐다.
- 라이브 HTML에서 GTM `GTM-T8FLZNT`가 확인됐다.
- 라이브 HTML에서 아임웹 채널톡 스크립트 `/js/channel_plugin.js`가 확인됐다.
- GTM 라이브 JS에 `2026-04-17-aibio-micro-events-v1` 마이크로 전환 리스너가 포함되어 있다.
- GTM 라이브 JS에 `aibio_channeltalk_open` GA4 태그와 Meta `trackCustom` 태그가 포함되어 있다.
- 2026-04-24 TJ님 캡처 기준, 채널톡 버튼 클릭 후 Meta Pixel Helper에서 `aibio_channeltalk_open`이 Active로 잡혔다.
- Pixel Helper 세부값은 `business_unit=aibio`, `interaction_type=channeltalk_open`, `source=gtm`, setup method `Manual`, pixel location `https://aibio.ai/`, frame `Window`, load time `7.76 ms`다.
- Events Manager 이벤트 목록에서 `aibio_channeltalk_open`은 활성, 통합 `브라우저`, 총 이벤트 4건, 최근 수신 2일 전으로 보인다.
- Events Manager 이벤트 소스 탭에서는 웹사이트/브라우저 기준으로 2026-04-20 2건, 2026-04-21 1건, 2026-04-22 1건이 보인다.
- TJ님 확인 기준 Meta 맞춤 이벤트 확인이 완료됐다. 따라서 `aibio_channeltalk_open`은 광고 기능에 연결할 수 있는 후보가 됐다.
- Meta 진단에는 "전환 API로 추적된 Meta 픽셀 이벤트의 비율을 개선하여 결과당 비용을 낮출 수 있다"는 권고가 보인다. 이 권고는 `aibio_channeltalk_open`이 아니라 `결제 시작` 같은 표준 이벤트의 서버 수신 비율을 높이라는 의미로 본다.
- Meta Ads Insights API에서 2026-04-17 ~ 2026-04-21 광고 row 5건을 확인했다. 이 기간 광고비는 643,802원이고, action type에는 `aibio_channeltalk_open` 또는 channeltalk 계열 이름이 아직 보이지 않았다.
- 같은 기간 Meta action에는 `lead`, `offsite_conversion.fb_pixel_lead`, `onsite_web_lead`가 각각 10건으로 보였다. `landing_page_view`와 `omni_landing_page_view`는 각각 3,978건으로 같은 값이라 합산하면 안 된다.
- 채널톡 공식 SDK는 `showMessenger`, `onShowMessenger`, `onChatCreated`, `track` 계열 동작을 제공한다.
- 바이오컴 CAPI 문서 기준으로, 브라우저 Pixel과 서버 CAPI를 같이 보낼 때는 같은 `event_id`가 필요하다.

### 완료/남은 것
- `AIBIO - ChannelTalk Open` Custom Conversion은 2026-04-24 API로 생성됐다. ID는 `26760275576970796`이다.
- `AIBIO - ChannelTalk Open 30d` Website Custom Audience는 2026-04-24 API로 생성됐다. ID는 `120244857451580036`, retention은 30일이다.
- 2026-04-25 TJ님 캡처 기준 광고관리자 컬럼에 `AIBIO - ChannelTalk Open`, `AIBIO - ChannelTalk Open 전환값`, `AIBIO - ChannelTalk Open당 비용`이 추가됐다.
- 현재 GTM 이벤트는 `채널톡 열림`까지다. 실제 상담방 생성이나 메시지 전송 완료는 아직 별도 이벤트로 분리되어 있지 않다.
- AIBIO 홈페이지에는 같은 Pixel ID로 PageView를 보내는 코드가 두 군데 보인다. 채널톡 이벤트가 중복 발화되는지 확인해야 한다.
- 서버 CAPI 경량 endpoint `/api/meta/capi/track`는 현재 허용 origin에 `https://aibio.ai`가 없다. AIBIO에서 CAPI mirror를 쓰려면 승인 후 CORS 허용 origin을 추가해야 한다.

### 머신러닝 제한됨 판단
현재 화면의 "머신러닝 제한됨"은 바꾸는 것보다 원인을 받아들이고 운영 기준을 조정하는 쪽이 맞다.

Meta 공식 도움말 기준으로 이 상태는 광고 세트가 학습 단계를 벗어날 만큼 충분한 결과를 만들지 못했다는 의미다. AIBIO 최근 30일 API 수치도 이 해석과 맞다.

| 항목 | 값 |
|---|---:|
| 기준 window | 2026-03-26 ~ 2026-04-24 |
| 기준 시각 | 2026-04-25 11:10 KST |
| source | Meta Marketing API v22.0 |
| 리드 캠페인 광고세트 | `120243904163210036` |
| objective | `OUTCOME_LEADS` |
| optimization_goal | `OFFSITE_CONVERSIONS` |
| 광고비 | 1,172,634원 |
| Lead | 13 |
| blended CPL | 약 90,203원 |
| 랜딩 페이지 조회 | 1,105 |
| 링크 클릭 | 1,530 |
| channeltalk action type | 0 |

학습 제한을 없애기 위해 단순히 예산을 올리면, 현재 blended CPL 기준으로 주 50 Lead를 만들려면 약 4,510,150원/주, 644,307원/일이 필요하다. 가장 성과가 좋은 `260401_리커버리랩 이벤트2` 기준 CPL 64,638원으로 계산해도 약 3,231,900원/주, 461,700원/일이 필요하다.

따라서 지금은 "제한됨" 문구를 없애려고 최적화 이벤트를 `landing_page_view`나 `aibio_channeltalk_open`으로 낮추지 않는다. `Lead` 최적화를 유지하고, 이벤트2 소재를 기준으로 랜딩/폼/상담 연결을 개선해 실제 Lead 수를 늘리는 쪽이 맞다.

### 채널톡 버튼의 위치
채널톡 버튼 클릭은 4단계다. 즉 `Lead` 전 단계다.

이 단계는 아래 용도로 유효하다.

- 채널톡을 연 사용자 리타겟팅
- 광고 소재별 상담 의도 비교
- 랜딩 CTA 위치와 문구 비교
- 폼 제출 이전 이탈 구간 분석

반대로 아래 용도로는 아직 부적절하다.

- Meta 리드 캠페인의 최종 최적화 이벤트
- 상담팀 실적 기준
- 방문 예약 수 대체 지표
- 센터 매출 ROAS 계산 지표

`aibio_channeltalk_open`을 광고세트 최적화 이벤트로 쓰려면 아래 조건이 먼저 필요하다.

1. 같은 클릭 1회에서 이벤트가 1회만 발생한다.
2. 자동 팝업이나 재방문 자동 열림이 전환으로 섞이지 않는다.
3. 최근 7일 기준 이벤트 수가 충분하다.
4. `aibio_channeltalk_open -> form_submit -> 상담/방문` 전환율이 확인된다.
5. Meta Events Manager에서 이벤트 품질 경고가 닫혀 있다.

이 조건이 닫히기 전에는 `Lead` 또는 내부 `form_submit`을 최적화 중심으로 유지한다.

## CAPI 관점

### 바이오컴 CAPI 문서에서 가져올 점
`meta/capimeta.md`의 핵심은 단순하다.

브라우저 Pixel만 쓰면 브라우저 차단, 쿠키 제한, 네트워크 실패로 일부 이벤트가 빠질 수 있다. 서버 CAPI를 같이 쓰면 보완할 수 있다. 단, 브라우저와 서버가 같은 행동을 동시에 보낼 때는 `event_id`와 이벤트명이 같아야 Meta가 중복을 줄인다.

이 원칙은 AIBIO에도 적용된다. 다만 채널톡 버튼 클릭은 아직 최종 전환이 아니므로, 처음부터 서버 CAPI까지 붙일 필요는 낮다.

### 2026-04-24 Meta 전환 API 권고 판단
Meta 진단의 "전환 API로 추적된 Meta 픽셀 이벤트의 비율을 개선" 권고는 맞는 방향이다. 하지만 지금 바로 모든 AIBIO 이벤트에 CAPI를 붙이라는 뜻은 아니다.

이 권고는 특히 `결제 시작`처럼 광고 최적화나 보고 정확도에 쓰이는 표준 이벤트에서 중요하다. 화면 기준 `결제 시작`은 여러 소스에서 수신되고, 이벤트 매칭 품질은 `7.7/10`, 총 이벤트는 22건이다. Meta가 말하는 개선 포인트는 브라우저 Pixel만 받은 이벤트 비율을 줄이고, 같은 이벤트를 서버 CAPI로도 받아 보고 정확도와 매칭 품질을 올리는 것이다.

현재 AIBIO 기준 판단은 아래다.

| 질문 | 판단 |
|---|---|
| 채널톡 open을 쓰기 위해 CAPI가 지금 필수인가 | 아니다. `aibio_channeltalk_open`은 4건 수준의 마이크로 전환이라 browser-only로 먼저 충분하다. |
| Meta 권고를 무시해도 되는가 | 아니다. `Lead`, `Schedule`, `InitiateCheckout`를 최적화 이벤트로 쓸 때는 CAPI mirror가 필요해진다. |
| 지금 바로 배포해야 하는가 | 아직 아니다. AIBIO origin 허용과 브라우저 snippet 변경이 필요하므로 보안/배포 변경 승인 대상이다. |
| 우선순위는 어디인가 | Custom Conversion/audience 생성 후, `Lead`와 방문 예약 원장이 안정화되면 CAPI를 붙인다. |

따라서 이 권고의 처리 순서는 `채널톡 custom event 활용 -> Lead/상담/방문 기준선 -> CAPI mirror`가 맞다.

### 현재 서버 코드 상태
로컬 코드 기준으로 `/api/meta/capi/track` 경량 endpoint는 이미 있다.

| 항목 | 현재 상태 |
|---|---|
| endpoint | `POST /api/meta/capi/track` |
| 허용 이벤트 | `ViewContent`, `AddToCart`, `InitiateCheckout`, `Lead`, `Search` |
| AIBIO Pixel 허용 | env 기본값에 `1068377347547682` 존재 |
| CORS 허용 origin | `biocom.kr`, `thecleancoffee.com`, `localhost:7010` 중심 |
| AIBIO origin | 현재 미포함 |

따라서 AIBIO에서 CAPI mirror를 쓰려면 아래 변경이 필요하다.

1. 승인 후 `FUNNEL_ALLOWED_ORIGINS`에 `https://aibio.ai`, `https://www.aibio.ai`, `https://aibio.imweb.me`를 추가한다.
2. 버튼 클릭을 CAPI로 보낼지, 실제 상담방 생성만 CAPI로 보낼지 결정한다.
3. 브라우저 Pixel과 CAPI가 같은 `event_id`를 쓰게 만든다.
4. Meta Test Events에서 browser/server dedup을 확인한다.

### CAPI를 붙이는 우선순위
| 이벤트 | CAPI 필요성 | 이유 |
|---|---|---|
| `aibio_channeltalk_open` | 낮음 | 버튼 열림은 마이크로 신호라 browser-only로 먼저 충분하다. |
| `aibio_channeltalk_chat_created` | 중간 | 실제 상담 시작이면 누락 보완 가치가 있다. |
| `Lead` | 높음 | 광고 최적화 핵심 이벤트다. 중복 방지와 품질 확인이 중요하다. |
| `InitiateCheckout` | 중간~높음 | Meta가 CAPI 커버율 개선을 권고한 표준 이벤트다. 다만 AIBIO의 주 전환이 구매가 아니므로 `Lead`와 `Schedule`보다 뒤에 둔다. |
| `Schedule` | 높음 | 방문 예약은 고품질 전환이다. 서버 원장 기준으로 보내야 한다. |

## 구현 판단

### 현재 GTM 구조
현재 GTM listener는 두 경로로 `aibio_channeltalk_open`을 만든다.

| 감지 경로 | 의미 |
|---|---|
| `ChannelIO('onShowMessenger')` | 채널톡 메신저가 실제로 열림 |
| `#ch-plugin`, `#ch-plugin-entry`, `[id^="ch-plugin"]` 클릭 | 채널톡 위젯 클릭 |

그리고 한 페이지에서 같은 이벤트가 여러 번 나가지 않도록 `window.__AIBIO_GTM_MICRO_aibio_channeltalk_open` 플래그로 중복을 막는다.

### 이 구조의 장점
- 실제 버튼이 어디에 렌더링되든 `id^="ch-plugin"` 계열로 잡을 수 있다.
- ChannelTalk SDK가 늦게 로드되어도 30회까지 바인딩을 재시도한다.
- GA4와 Meta가 같은 `dataLayer` 이벤트를 기준으로 움직인다.
- Meta에는 custom event로 보내므로 `Lead` 전환을 오염시키지 않는다.

### 보완할 점
현재 이벤트 이름은 `open`이다. 순수하게 버튼 클릭만 보고 싶으면 아래처럼 분리하는 것이 더 정확하다.

| 이벤트 | 발화 조건 | 쓰임 |
|---|---|---|
| `aibio_channeltalk_button_click` | 채널톡 위젯 직접 클릭 | 버튼 CTA 성과 |
| `aibio_channeltalk_open` | 메신저 실제 열림 | 상담 채널 진입 |
| `aibio_channeltalk_chat_created` | 새 상담방 생성 | 실제 상담 시작 |

현재는 `button_click`과 `open`을 하나로 묶고 있다. 운영 초기에는 괜찮지만, 광고 최적화에 쓰려면 분리하는 편이 낫다.

## 위험과 주의

### 위험 1. 버튼 클릭을 Lead로 오해
채널톡 버튼 클릭은 연락처 제출이 아니다. 이 이벤트를 `Lead`로 보내면 Meta 리드 수가 부풀 수 있다.

대응: `aibio_channeltalk_open`은 custom event로 유지한다. 실제 폼 제출 또는 상담 신청 완료만 `Lead`로 보낸다.

### 위험 2. 자동 열림과 직접 클릭 혼합
`onShowMessenger`는 사용자가 직접 클릭했을 때뿐 아니라 코드나 자동 동작으로 메신저가 열릴 때도 실행될 수 있다.

대응: 순수 버튼 클릭이 필요하면 `aibio_channeltalk_button_click`을 별도 이벤트로 분리한다.

### 위험 3. Pixel 중복 설치
라이브 HTML에는 같은 Pixel ID `1068377347547682`로 PageView를 보내는 코드가 두 구간에서 보인다.

대응: 우선 PageView 중복과 채널톡 custom event 중복을 Pixel Helper에서 확인한다. 중복이 있으면 수동 Pixel 코드와 아임웹 내장 Pixel 중 하나를 정리한다.

### 위험 4. 민감 정보 전송
AIBIO는 건강, 회복, 컨디션 개선 맥락의 서비스다. Meta로 증상, 질환, 상담 메시지 본문, 전화번호, 이메일 원문을 보내면 안 된다.

대응: Meta에는 이벤트명, URL, generic parameter만 보낸다. 고객 식별자는 필요할 때만 해시 또는 Meta가 허용하는 형식으로 처리한다. 상담 내용은 내부 장부에만 둔다.

### 위험 5. CAPI 중복
나중에 CAPI mirror를 켜면 browser event와 server event가 둘 다 Meta에 도착한다.

대응: 같은 행동은 같은 `event_id`와 같은 이벤트명으로 보낸다. Meta 공식 문서 기준으로 Pixel의 `eventID`와 CAPI의 `event_id`가 맞아야 중복 제거가 된다.

- 폼 제출은 이제 Meta에서 `Lead`로 잡을 수 있다. 실제로 리드 캠페인에서 `lead`, `onsite_web_lead`, `offsite_conversion.fb_pixel_lead`가 보고된다.
- 트래픽 캠페인에서도 기술적으로는 광고 클릭 후 폼 제출이 발생하면 `Lead` action으로 보고될 수 있다. 하지만 최근 30일 API 기준으로는 트래픽 캠페인에 Lead가 0이다.
- 채널톡 클릭은 `AIBIO - ChannelTalk Open` Custom Conversion과 30일 audience로 참조할 수 있다. 다만 최근 30일 광고 인사이트에는 아직 0으로 나온다.
- 채널톡 클릭을 `Lead`로 승격하지 않는다. 리타겟팅/소재 진단/랜딩 CTA 개선 신호로 먼저 쓴다.

### 현재 GTM 이벤트 흐름
```text
ChannelTalk button click 또는 ChannelIO onShowMessenger
  -> dataLayer.push({ event: "aibio_channeltalk_open", ... })
  -> GA4 aibio_channeltalk_open
  -> Meta trackCustom aibio_channeltalk_open
```

### 현재 Meta Lead 흐름
```text
AIBIO form submit
  -> dataLayer.push({ event: "aibio_form_submit", ... })
  -> GA4 generate_lead
  -> Meta Lead
```

### CAPI 확장 후보 흐름
```text
ChannelTalk chat_created 또는 Lead 확정
  -> browser Pixel event with eventID
  -> POST /api/meta/capi/track with same event_id
  -> Meta dedup by event_name + event_id
```

### 금지 payload
Meta로 아래 값은 보내지 않는다.

| 값 | 이유 |
|---|---|
| 상담 메시지 본문 | 민감 정보 가능성 |
| 증상, 질환, 건강 고민 | 민감 정보 가능성 |
| 전화번호 원문 | 개인정보 |
| 이메일 원문 | 개인정보 |
| 상담사가 남긴 내부 메모 | 운영 내부 정보 |

### 검증 기준
- wiki 링크 검증: `python3 scripts/validate_wiki_links.py aibio/채널톡퍼널.md`
- 이벤트 검증: 한 번의 채널톡 클릭에 `aibio_channeltalk_open` 1회
- 전환 검증: 채널톡 클릭만으로 `Lead`가 발화하지 않음
- CAPI 검증: server mirror를 켤 경우 browser/server event_id 일치
