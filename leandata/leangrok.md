**린 분석(Lean Analytics) 책 조사 결과입니다.**

원서 제목은 **Lean Analytics: Use Data to Build a Better Startup Faster** (Alistair Croll, Benjamin Yoskovitz 지음, O'Reilly, 2013)이며, 한국어판(한빛미디어, 2014, 번역: 위선주)은 부제 **"성공을 예측하는 31가지 사례와 13가지 패턴"**으로 출간됐습니다. 이 책은 에릭 리스의 《린 스타트업》 시리즈 3탄으로, **"측정(Measure)"** 단계에 초점을 맞춰 데이터로 스타트업(또는 기존 기업)의 리스크를 줄이고 빠르게 성장시키는 실전 가이드입니다. 100명 이상의 창업자·투자자 인터뷰와 **30개 이상(책 홍보상 31가지로 표현)**의 실제 사례 연구를 바탕으로 작성됐어요.

한국어 부제의 **"31가지 사례"**는 책 전체 **31장** 구성(4부)과 산재된 30+ 사례 연구를 가리키는 마케팅 표현입니다. **"13가지 패턴"**은 공식 무료 e-book **"Analytics Lessons Learned"**에 실린 **정확히 13개 핵심 사례 연구**를 의미합니다(저자 공식 사이트 leananalyticsbook.com에서 무료 다운로드 가능). 이 13개가 성공 예측의 **실전 패턴(데이터 기반 가설 검증·피벗·최적화 원칙)**을 보여주는 대표 예시예요.

### 책 핵심 프레임워크 (성공 예측의 기본 패턴)
책은 단순 사례 나열이 아니라 체계적인 틀을 제시합니다. 주요 패턴은 다음과 같아요:

- **One Metric That Matters (OMTM)**: 한 번에 **하나의 가장 중요한 지표**만 집중. 현재 사업 단계에서 가장 큰 리스크를 해결하는 지표를 골라 행동하게 함.
- **5단계 성장 모델 (Empathy → Stickiness → Virality → Revenue → Scale)**: 각 단계별 리스크와 OMTM이 바뀜. 목표(게이트)를 넘어야 다음 단계로.
  - Empathy: 고객 문제 검증 (인터뷰, MVP)
  - Stickiness: 제품 유지력 (활성 사용자, engagement)
  - Virality: 바이럴 성장 (K-factor, cycle time)
  - Revenue: 수익화 (LTV, CAC)
  - Scale: 확장 (채널 최적화, 경쟁 대응)
- **6가지 사업 모델**: E-commerce, SaaS, Free Mobile App, Media Site, User-Generated Content, Two-Sided Marketplaces. 각 모델별 OMTM과 벤치마크가 다름.
- **Lines in the Sand (기준선)**: "이 정도면 성공인가?" 판단하는 구체적 숫자 벤치마크 (Part 3). 예시:
  - E-commerce: Conversion rate 평균 2% (좋음 10%), 장바구니 이탈 65% 이상 예상.
  - SaaS: Monthly churn 5% 이하 (최고 1.5~3%), CAC < LTV/3.
  - Mobile App: DAU/MAU 비율 (Fred Wilson 30/10/10 규칙 등).
  - Media: CTR 0.5~2% 등.

이 프레임워크들이 **"13가지 패턴"의 기반**이며, 데이터로 가설을 빠르게 테스트하고 피벗하며 벤치마크를 넘어가는 **성공 예측 패턴**입니다. (책은 vanity metric 피하기, cohort 분석, leading/lagging metric 구분 등도 자세히 다뤄요.)

### 13가지 사례 연구 + 성공 패턴 (무료 e-book 핵심 내용)
이 13개가 한국어 부제의 **"13가지 패턴"**에 해당하는 대표 사례입니다. 각 사례는 **"어떻게 데이터로 가설을 검증하고 피벗·최적화해서 성공을 예측/달성했는가"**를 보여줍니다. (영문 PDF 원문 기반 요약, 한국어로 번역·정리)

1. **Airbnb Photography (프로 사진 서비스)**: "프로 사진이 예약을 늘릴까?" 가설 테스트 (Concierge MVP). 결과: 사진 있는 listing이 2~3배 예약 증가. shoots/month 지표로 확장 → 수천 건 촬영 규모로 성장. **패턴**: 저비용 MVP로 빠른 가설 검증 + 성장 내 성장.
   
2. **Circle of Moms (모마 커뮤니티 피벗)**: 일반 소셜 앱에서 engagement 데이터 분석 → 엄마 사용자들이 메시지 50% 길고, 뉴스피드 클릭 180% 높음 발견. 피벗 후 450만 활성 사용자 성장. **패턴**: 데이터 마이닝으로 숨겨진 세그먼트 발견 + 과감 피벗.

3. **SEOmoz (Net Adds 집중)**: 수많은 KPI 대신 "Net Adds(신규 가입 - 해지)" 하나만 집중. 투자자 조언으로 KPI 최소화 → 조직 집중력 ↑. **패턴**: vanity metric 버리고 OMTM 하나로 팀 정렬.

4. **Backupify (소비자 → B2B 피벗)**: 초기 방문자·트라이얼 지표 → MRR(월 반복 수익)으로 진화. 소비자 CAC 너무 높아 B2B 피벗 → CLV:CAC 비율 5~6배 달성. **패턴**: 사업 규모에 따라 metric 자연 진화 + 수익성 검증.

5. **ClearFit (가격 모델 변경)**: $99/월 구독 → 고객 이해 부족. $350/건당 요금으로 변경 → 매출 10배 성장, 월 30% 증가. **패턴**: 고객 구매 행동에 맞춘 가격 실험 + quantitative 검증.

6. **LikeBright (Mechanical Turk 인터뷰)**: TechStars 거절 후, Mechanical Turk로 4시간 만에 100명 여성 인터뷰 → 고객 이해 입증, 입성 성공. **패턴**: Empathy 단계에서 저비용 툴로 빠른 qualitative 데이터 수집.

7. **Sincerely (모바일 CAC 문제)**: 모바일 광고 CAC 너무 높고 추적 어려움. 고가 앱(Ink Cards 등)으로 피벗 + virality 강조 → LTV 30% ↑. **패턴**: CAC/LTV/payback 추적 + 바이럴 채널 전환.

8. **Parse.ly (소비자 → 퍼블리셔 피벗)**: 소비자 뉴스 리더(성장만, 수익 없음) → 2번 피벗(퍼블리셔 분석 툴). signup → conversion 지표로 변경. **패턴**: 수익화 가설 조기 테스트 + 시장 피벗.

9. **WineExpress (A/B 테스트)**: "Wine of the Day" 페이지 레이아웃 최적화 (비디오 위로 배치) → visitor당 수익 41% 증가. **패턴**: revenue per visitor 같은 본질 metric 최적화 (단순 conversion 아님).

10. **OfficeDrop (paid churn 집중)**: paid churn(유료 해지) <4% 목표로 관리. 브라우저 → 데스크톱/모바일 앱 피벗 → churn ↓ + 성장. **패턴**: churn이 건강 지표 + engagement와 연계.

11. **Kingsway Church (비영리 적용)**: velocity(새 이니셔티브 속도) + vision virality(비전 전파) 지표 추적. 신규 장소 commitment %로 확장 판단 → 교회 2배 성장. **패턴**: 스타트업 외(비영리·기업)에도 Lean Analytics 적용 가능.

12. **Swiffer (P&G, 사용자 관찰)**: 청소액 화학 개선에 매몰 → 실제 사용자 관찰(걸레 없이 닦는 행동)로 피벗 → Swiffer 도구 신제품, 5억 달러 카테고리 창출. **패턴**: 내부 가정 대신 고객 행동 관찰로 혁신 (intrapreneur 사례).

13. **EMI (데이터 문화 구축)**: 빅데이터 대신 100만 건 설문으로 소비자 취향 파악 → 아티스트 마케팅 맞춤. **패턴**: 단순·맥락 있는 데이터부터 시작해 조직 데이터 문화 조성.

이 13개 사례가 **성공 예측 패턴**의 본질을 보여줍니다: **가설 → 실험(MVP) → 측정(OMTM) → 피벗/최적화** 반복. 데이터로 리스크를 정량화하고, 벤치마크(Lines in the Sand)를 넘어가는 과정이 핵심이에요.

### 추가 사례 (본책 30+ 중 대표 예시)
책 본문에는 위 13개 외에도 Etsy(마켓플레이스 metric 관리), Facebook(사진 업로드 funnel 최적화), Houston Airport(수하물 대기 시간 인식 개선), HighScore House(활성 사용자 정의) 등 더 많은 사례가 흩어져 있습니다. 각 장마다 실전 적용 예시가 있어요.
