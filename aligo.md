## 실패 원인 분석

### `rslt: M` — 템플릿을 찾을 수 없음

카카오 비즈메시지에서 템플릿을 매칭하지 못할 때 발생. 가능한 원인:

1. **`dormant=Y` (휴면)** ← **이번 원인**
   - 카카오는 일정 기간(약 1년) 미사용 템플릿을 자동 휴면 처리
   - 알리고 API에서 `dormant=Y`인 템플릿은 발송해도 카카오에서 거부
   - 해결: 알리고 관리자 페이지에서 휴면 해제 신청, 또는 활성(`dormant=N`) 템플릿만 사용

2. **본문 불일치 (exact-match 실패)**
   - 카카오는 승인된 템플릿 본문과 한 글자도 다르면 거부
   - 공백, 줄바꿈, 특수문자 차이도 실패 원인
   - 해결: `render-preview` API로 변수 치환 후 `exactMatch: true` 확인

3. **버튼 구조 불일치**
   - 버튼 JSON의 키 이름(`linkType` vs `type`)이 다르면 실패
   - 해결: 알리고가 기대하는 버튼 포맷 정확히 맞추기
