**서버 사이드 추적(Server-Side Tracking)**, 특히 GA4의 **Measurement Protocol**을 이용한 방식은 브라우저(클라이언트)를 거치지 않고 **우리 서버가 직접 구글 서버로 데이터를 쏘는 방식**입니다.
네이버페이처럼 사용자가 우리 사이트를 떠나 결제하고 돌아오지 않을 수도 있는 상황에서 '데이터 유실'을 막는 가장 강력한 무기죠. 어떻게 구현하는지 단계별로 상세히 설명해 드릴게요.

1. 서버 사이드 추적의 기본 원리
브라우저 방식(Client-side)은 사용자가 '구매 완료 페이지'를 봐야만 코드가 실행되지만, 서버 사이드 방식은 **네이버페이로부터 결제 완료 통보(Webhook)를 받는 즉시** 우리 서버가 GA4에 보고합니다.
**핵심 매커니즘: "연결 고리(ID) 저장"**
서버는 사용자가 누군지 모릅니다. 그래서 사용자가 결제 버튼을 누르는 순간, 브라우저에 있는 **GA4 식별값(Client ID, Session ID)**을 서버 DB에 주문 번호와 함께 미리 저장해 두는 것이 핵심입니다.

2. 구현 5단계 (기술적 워크플로우)
**Step 1. GA4 식별값 추출 및 전송**
사용자가 결제 버튼을 클릭할 때, 브라우저의 쿠키에서 아래 두 값을 추출해 결제 요청 정보와 함께 자사 서버로 보냅니다.

- **Client ID (**`**_ga**` **쿠키):** 기기 식별값
- **Session ID (**`**_ga_<container-id>**` **쿠키):** 현재 세션 식별값 (이게 있어야 UTM 소스가 유지됨)

**Step 2. 서버 DB에 임시 저장**
결제가 진행되는 동안 서버는 `주문번호 : Client ID : Session ID`를 매칭하여 DB에 보관합니다.
**Step 3. 네이버페이 결제 완료 통보(Webhook) 수신**
사용자가 네이버 페이지에서 결제를 마치면, 네이버페이 서버는 자사몰 서버로 "이 주문번호 결제 완료됐어!"라는 신호(Webhook)를 보냅니다. **이때 사용자가 창을 닫았어도 서버 간 통신은 일어납니다.**
**Step 4. GA4 Measurement Protocol 호출**
우리 서버는 보관해둔 ID 정보를 꺼내 GA4 서버로 HTTP POST 요청을 보냅니다.

- **Endpoint:** `[https://www.google-analytics.com/mp/collect?measurement_id=G-XXXXX&api_secret=XXXXX](https://www.google-analytics.com/mp/collect?measurement_id=G-XXXXX&api_secret=XXXXX)`
- **Payload (데이터 예시):**
- JSON




{
  "client_id": "1234567.890123", // DB에서 꺼낸 값
  "events": [{
    "name": "purchase",
    "params": {
      "session_id": "168000000", // DB에서 꺼낸 값 (UTM 유지의 핵심)
      "transaction_id": "ORDER_12345",
      "value": 50000,
      "currency": "KRW",
      "items": [{ "item_id": "SKU_001", "item_name": "멋진 상품" }]
    }
  }]
}

**Step 5. GA4 데이터 매칭**
GA4는 서버로부터 받은 `client_id`와 `session_id`를 보고, "아, 아까 광고 보고 들어왔던 그 사람이네!"라고 인지하여 **기존 UTM 소스에 구매 데이터를 합쳐줍니다.**

3. 이 방식의 압도적인 장점

4. **결제 이탈 무시:** 고객이 결제 후 네이버 페이지에서 창을 꺼버리거나, 폰을 던져버려도(?) 매출 데이터는 100% 들어옵니다.
5. **광고 성과(ROAS) 정확도 향상:** UTM 소스가 유실되지 않으므로 어떤 광고가 돈을 벌어다 줬는지 정확히 알 수 있습니다.
6. **광고 차단기(Ad-Block) 우회:** 브라우저 스크립트가 아니므로 광고 차단 프로그램의 영향을 받지 않습니다.



7. 주의사항 (이것 안 하면 말짱 도루묵)

> **"API Secret" 발급 필수**
> GA4 관리 > 데이터 스트림 > [스트림 선택] > **Measurement Protocol API 비밀번호**에서 API 키를 생성해야 서버에서 데이터를 보낼 권한이 생깁니다.
> **"Session ID"를 반드시 포함하세요**
> 많은 개발자가 `client_id`만 보내는 실수를 합니다. 이 경우 구매 데이터는 잡히지만, **'직접(Direct)' 유입**으로 잡혀서 UTM 성과 측정이 안 됩니다. 반드시 `session_id`를 함께 쏴야 합니다.



**정리하자면**
서버 사이드 추적은 **"브라우저의 입을 빌리지 않고, 서버가 직접 GA4에게 비밀 보고를 하는 시스템"**을 만드는 것입니다.
