# 쿠폰명 API 연동 및 백필 결과

작성일: 2026-04-10

## 한 줄 결론

쿠폰명 연동과 백필은 **최근 로컬 아임웹 주문 캐시에 들어 있는 발행쿠폰코드 기준으로는 100% 완료**됐다.

다만 전체 쿠폰 분석이 100% 끝났다고 보면 안 된다. 운영 주문 원장 전체 기간에는 쿠폰 할인금액은 있지만 발행쿠폰코드가 없어서, 2024-11-04부터 2026-04-09까지의 모든 주문에 쿠폰명을 정확히 붙이는 작업은 아직 별도 보강이 필요하다.

## 무엇을 연동했나

운영 주문 원장에는 `coupon_discount`처럼 할인금액은 있지만, 사람이 읽을 수 있는 쿠폰명이 없다.

최근 로컬 아임웹 주문 캐시에는 `use_issue_coupon_codes`가 들어 있다. 이 값은 사람이 읽는 쿠폰명이 아니라 아임웹 내부 발행쿠폰 ID다.

이번 작업에서는 이 내부 발행쿠폰 ID를 아임웹 쿠폰 API로 조회해서 쿠폰명으로 바꿔 로컬 SQLite에 백필했다.

사용한 API:

- 쿠폰 마스터 목록: `GET /v2/shop/coupons`
- 발행쿠폰 단건 조회: `GET /v2/shop/issue-coupons/:issueCode`

추가된 로컬 테이블:

- `imweb_coupon_masters`
- `imweb_issue_coupons`

추가된 백엔드 엔드포인트:

- `POST /api/crm-local/imweb/sync-coupons`
- `GET /api/crm-local/imweb/coupon-stats?site=...`

## 화면 반영 내용

`/coupon` 화면에 다음 내용을 반영했다.

- 상품 카테고리 표에 대표 쿠폰명 추가
- 쿠폰명 백필 기간과 전체 고객 기준 분석 기간이 다르다는 주석 추가
- 기존 “샘플 50개 성공” 문구를 실제 백필 결과로 교체
- 기존 “더클린커피 쿠폰 0건/미연동” 문구를 로컬 캐시 기준 실제 백필 수치로 교체

현재 화면에서 중요한 해석:

- 바이오컴 카테고리별 고객 기준 쿠폰 사용률은 여전히 운영 주문 원장 전체 기간 기준이다.
- 카테고리 옆 대표 쿠폰명은 최근 로컬 아임웹 캐시와 운영 주문 원장을 주문번호로 조인해 본 참고값이다.
- 따라서 대표 쿠폰명은 “최근 캐시 기준 대표 패턴”으로 봐야 하고, 2024-11-04부터 2026-04-09까지 전체 기간의 확정 쿠폰명 분포로 보면 안 된다.

## 100% 완료 여부

완료라고 봐도 되는 범위:

- 아임웹 쿠폰 API 연동 가능성 확인: 완료
- 로컬 아임웹 주문 캐시에 존재하는 고유 발행쿠폰코드 쿠폰명 백필: 완료
- biocom 고유 발행쿠폰코드 `2,238개` 매핑: 완료
- thecleancoffee 고유 발행쿠폰코드 `416개` 매핑: 완료
- 매핑 결과를 조회하는 백엔드 엔드포인트: 완료
- `/coupon` 화면의 대표 쿠폰명 표시와 설명 문구 정리: 완료

아직 100%가 아닌 범위:

- 운영 주문 원장 전체 기간(`2024-11-04 ~ 2026-04-09`)의 모든 주문에 쿠폰명을 붙이는 것
- 주문 내 상품 라인아이템별로 “이 쿠폰이 정확히 어느 상품에 적용됐는지”를 확정하는 것
- 매일 자동으로 쿠폰 마스터와 발행쿠폰을 동기화하는 스케줄러 고정
- 쿠폰별 ROI를 전체 매출·고객·재구매 기준으로 자동 산출하는 대시보드화

## 왜 아직 전체 100%가 아닌가

운영 주문 원장에는 상품명, 주문번호, 쿠폰 할인금액은 있지만 아임웹 발행쿠폰 ID가 없다.

반대로 로컬 아임웹 주문 캐시에는 발행쿠폰 ID가 있지만, 현재 캐시 범위가 최근 구간에 한정되어 있고 주문 단위 중심이다.

따라서 전체 기간의 카테고리별 쿠폰명을 확정하려면 아래 중 하나가 더 필요하다.

- 아임웹 주문 API로 과거 주문을 더 길게 백필해서 `use_issue_coupon_codes`를 확보
- 아임웹 발행쿠폰 목록을 쿠폰 마스터별로 더 넓게 조회한 뒤 주문번호와 운영 원장을 조인
- 가능하면 아임웹 주문 상세 또는 상품 라인아이템까지 포함해 주문 단위 쿠폰을 상품 단위로 배분
