### 2-2. imweb_status 의미 섞임

피드백 `§주의할 점 3` 반영. 아래를 기록한다.

> **백필 713 rows의 `imweb_status` 값은 실제로는 운영 `tb_iamweb_users.payment_status`(한글: `거래종료`, `거래개시` 등) 출처다.** 기존 아임웹 API 싱크 row의 `imweb_status`(영문 코드)와 의미 체계가 다르다. 이 row를 대시보드에서 카운트할 때 `order_key LIKE 'iamweb_%'` 필터로 백필분을 구분하거나, `raw_json._source` 값을 참고하면 된다.

- 기간 + 상품 조합 매칭이지만 일반 유입 고객이 섞였을 가능성(2-10~11 주문량은 전주 평균과 차이 없음, spike 없음)
- 쿠폰/UTM/landing 자동 신호 0건
- 회사 엑셀 주문번호 미제공

`classifyCoopOrder` helper에서 `allowlist` 신호는 confidence 1.0으로 판정하지만, 이번 allowlist는 추론 기반이라 마스터 metadata `classification_confidence: 0.4`를 별도로 기록해 운영 판단 시 참고하게 했다.
